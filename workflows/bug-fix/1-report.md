# Bug Fix - STEP 1: Report（バグ報告）

**担当**: Kai（PM）
**入力**: ユーザーからのバグ報告

---

## 🎯 目的

「再現可能・再発防止可能」なバグ報告に整理する。

---

## 📋 バグレポートの必須項目

```markdown
# Bug #XXX: [短い説明]

## 概要
（1〜2文でバグの内容）

## 環境
- ブラウザ/OS:
- バージョン:
- ユーザーの権限:
- 再現環境: dev / staging / production

## 再現手順
1. 〇〇ページを開く
2. △△をクリック
3. □□と入力
4. □□が起こる ← バグ

## 期待動作
本来こうなるべき

## 実際の動作
こうなってしまう

## 影響範囲
- 影響を受けるユーザー数:
- ビジネス影響:
- データ破損の可能性: あり / なし

## 重要度
- P0: 即対応（サービス停止）
- P1: 24時間以内（主要機能不全）
- P2: 1週間以内（機能不全だが回避策あり）
- P3: 計画的対応

## スクリーンショット / ログ
（添付）

## 関連情報
- 関連Issue:
- 関連PR:
- 直近のリリース:
```

---

## ✅ 完了条件

- [ ] 再現手順が他人にも実行可能
- [ ] 期待動作が要件書/設計書から確認できる
- [ ] 影響範囲が定量的に把握できている
- [ ] 重要度が決定している

→ 次は STEP 2: 解析（`workflows/bug-fix/2-analyze.md`）
