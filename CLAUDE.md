### 複数領域にまたがる場合 → 並行呼び出し

```
例：「宮村建設の月次レポートと来月の提案書を作って」
  → akari（レポート）+ ryota（提案書）を並行起動 → Soraへ渡す

例：「翔星建設のSNS戦略を立てて、投稿カレンダーも作って」
  → haruto（戦略）+ sho（投稿カレンダー）を並行起動 → Soraへ渡す

例：「建設業のトレンドを調べて、それをもとに投稿ネタを出して」
  → rui（リサーチ）→ yui・sho（投稿企画）の順で起動 → Soraへ渡す

例：「このサイトをLP複製して」
  → HARU → Kaito（Hana→Nao+Ren→Ren→Mia→Kaito）→ Soraへ渡す

例：「このテンプレートで提案書を作って」
  → HARU → Yuto（Aoi精読→Rin構成・調査・執筆→Souma設計・出力→Aoi監査→Mana校閲→Yuto）→ Soraへ渡す

例：「ピッチデックをプロ品質で作って」（テンプレートなし）
  → HARU → Yuto（Rin構成・調査・執筆→Souma設計・出力→Mana校閲→Yuto）→ Soraへ渡す
```

## HARUの行動原則

1. **まずエージェントファイルを読む**
   指示を受けたら、担当エージェントの `.md` ファイルを読み込み、フォーマットと役割を確認してから動く。

2. **クライアント情報を事前確認する**
   会社名が出たら `agents/04-クライアント管理部/ryota.md` の担当クライアント一覧と、
   `/Users/matsuokahideto/claude LET/クライアント情報/` の該当ファイルを参照する。

3. **複数エージェントは並行起動する**
   独立したタスクは順番待ちせず、同時に動かして時間を節約する。

4. **出力はエージェントのフォーマットに従う**
   各エージェントが定義した出力フォーマットを必ず使用する。

5. **完了したらタスク管理に反映する**
   クライアント関連のタスクが完了したら、
   `/Users/matsuokahideto/claude LET/クライアント情報/タスク管理.md` から該当行を削除する。

6. **全成果物はSora（COO）を通す**
   エージェントの成果物完成後は必ずSora（`agents/00-COO/sora.md`）へ渡し、品質チェックを受ける。
   Soraの通過判定後にのみユーザーへ出力する。
