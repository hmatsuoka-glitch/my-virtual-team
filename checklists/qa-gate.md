# QA Gate Checklist（QA 通過判定）

**担当**: Mio（QA）
**使用タイミング**: STEP 5 QA時

このチェックリストで判定する：**PASS / FAIL / CONDITIONAL_PASS**

---

## 🔍 機能テスト

### 受け入れ基準の検証

要件書の Given-When-Then を全てテストする：

- [ ] 全機能の正常系をE2Eテスト
- [ ] 全機能の異常系をE2Eテスト
- [ ] 境界値テスト（最小値・最大値・空・1個・大量）

### ユーザーフローテスト

主要なユーザーシナリオを通しでテスト：

- [ ] 新規ユーザーがゴールに到達できる
- [ ] エラーから復帰できる（404 / 401 / 500 等）
- [ ] ログイン → 操作 → ログアウトの一連の流れ

## ⚡ パフォーマンステスト

- [ ] ページ読み込み速度（Lighthouse Score）
  - [ ] Performance: 80+
  - [ ] Accessibility: 90+
  - [ ] Best Practices: 90+
  - [ ] SEO: 80+
- [ ] API レスポンス時間（要件書の数値目標を満たす）
- [ ] 同時接続テスト（要件書の同時ユーザー数まで）
- [ ] バンドルサイズ（前リリース比 +10%以内）

## 🔐 セキュリティテスト

- [ ] OWASP Top 10 チェック
  - [ ] Injection（SQLi / XSS / Command）
  - [ ] 認証・セッション管理
  - [ ] アクセス制御
  - [ ] セキュリティ設定ミス
- [ ] 認証なしで認証必須エンドポイントにアクセス → 401
- [ ] 他ユーザーのデータにアクセス試行 → 403
- [ ] CSRF トークン検証
- [ ] レート制限の動作確認

## ♿ アクセシビリティテスト

- [ ] axe DevTools エラーゼロ
- [ ] キーボードのみで全操作可能
- [ ] スクリーンリーダー（VoiceOver / NVDA）で読める
- [ ] コントラスト比 WCAG AA 達成

## 📱 マルチデバイステスト

- [ ] iPhone（Safari）
- [ ] Android（Chrome）
- [ ] iPad
- [ ] Windows（Chrome / Edge）
- [ ] Mac（Safari / Chrome）

## 🌐 ブラウザ互換性

- [ ] Chrome 最新 + 1つ前
- [ ] Safari 最新 + 1つ前
- [ ] Edge 最新
- [ ] Firefox 最新

## 🧪 テストの品質

実装されたテストコード自体の品質：

- [ ] テストカバレッジ 80%+
- [ ] フレーキー（不安定）テストがない
- [ ] テスト実行時間が妥当（CI < 10分）
- [ ] テストが「実装の真似」ではなく「振る舞い」を検証

## 📊 監視・ログ

- [ ] エラー監視（Sentry等）が動作
- [ ] 重要操作のログが出ている
- [ ] ログに機密情報が出ていない

## 📝 ドキュメント

- [ ] README が最新
- [ ] API ドキュメントが最新
- [ ] CHANGELOG に変更内容が記載
- [ ] マイグレーション手順がある（DB変更時）

## 🚀 デプロイ準備

- [ ] staging で動作確認済み
- [ ] DB マイグレーションが本番で実行可能
- [ ] ロールバック手順を文書化
- [ ] 環境変数が本番に設定済み
- [ ] 監視アラートが設定済み

---

## 🎯 判定基準

### ✅ PASS（全項目クリア）
→ Sora（COO）へエスカレーション

### ⚠️ CONDITIONAL_PASS（軽微な問題のみ）
- 機能には影響しない問題
- リリース後に修正可能
- 例: コメント追加漏れ、テスト追加余地
→ Issue として登録 + リリース可能

### ❌ FAIL（重大な問題あり）
- 機能不全 / セキュリティ脆弱性 / パフォーマンス未達
→ 該当エージェントへ差し戻し（Issue付きで）
