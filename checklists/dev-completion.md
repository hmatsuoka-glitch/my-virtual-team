# Dev Completion Checklist（開発完了チェック）

**担当**: 各実装エージェント（riku / ao / kuu）
**使用タイミング**: 各タスク完了時・PR作成前

このチェックリストを全て満たさないと PR を出してはいけない。

---

## 📝 機能実装

- [ ] 設計書通りに実装した
- [ ] 受け入れ基準（Given-When-Then）を全て満たす
- [ ] エラーハンドリングが全ての非同期処理に存在
- [ ] 入力検証（Zod等）が全エンドポイントに存在
- [ ] エッジケースを考慮（null / 空 / 境界値 / 異常系）

## 🧪 テスト（TDD準拠）

- [ ] テストを実装より先に書いた（RED → GREEN → REFACTOR）
- [ ] ユニットテスト カバレッジ 80%以上
- [ ] エッジケースのテストがある
- [ ] エラーケースのテストがある
- [ ] スキップ（`it.skip`）/TODO（`it.todo`）が残っていない
- [ ] テストが CI で緑

## 🎨 コード品質

- [ ] TypeScript strict mode で型エラーゼロ
- [ ] ESLint エラーゼロ
- [ ] Prettier フォーマット適用
- [ ] 関数 20 行以内（例外は理由をコメント）
- [ ] 引数 3 つ以内（オブジェクト引数で集約）
- [ ] ネスト 3 段以内
- [ ] 1 ファイル 300 行以内
- [ ] マジックナンバー禁止（定数化）
- [ ] 命名が分かりやすい（略語・1文字変数禁止）

## 📚 ドキュメント

- [ ] 公開 API に JSDoc コメント
- [ ] 複雑なロジックに「なぜ」のコメント
- [ ] README.md に新機能の使い方追加（必要なら）
- [ ] 環境変数を追加した場合 .env.example も更新

## 🔐 セキュリティ

- [ ] ユーザー入力を全て検証
- [ ] SQL Injection 対策（ORM/Prepared Statement使用）
- [ ] XSS対策（Reactのエスケープ + DOMPurify）
- [ ] 認証・認可が必要な箇所に実装されている
- [ ] シークレットがコードにハードコードされていない
- [ ] 機密情報がログ出力されていない

## ⚡ パフォーマンス

- [ ] N+1 クエリがない（DBアクセス最適化）
- [ ] 不要な再レンダリングがない（React.memo / useMemo）
- [ ] バンドルサイズへの影響を確認
- [ ] 画像最適化（next/image等）

## ♿ アクセシビリティ

- [ ] セマンティックHTML（適切なタグ）
- [ ] ARIA 属性が必要箇所に存在
- [ ] キーボード操作可能
- [ ] スクリーンリーダーで読める
- [ ] フォーカス管理（モーダル等）
- [ ] コントラスト比 WCAG AA 以上

## 📱 レスポンシブ

- [ ] PC（>1024px）で動作確認
- [ ] Tablet（768-1024px）で動作確認
- [ ] Mobile（<768px）で動作確認

## 🌐 ブラウザ対応

- [ ] Chrome 最新版
- [ ] Safari 最新版
- [ ] Edge 最新版
- [ ] Firefox 最新版（推奨）

## 🔄 Git・CI/CD

- [ ] ブランチ名が規約通り（feature/fix/...）
- [ ] コミットメッセージが Conventional Commits 形式
- [ ] PR 説明文が必要事項を網羅
- [ ] CI が全て緑（lint / type / test / build）
- [ ] コンフリクトが解消されている

## 🤝 レビュー準備

- [ ] セルフレビューを実施した
- [ ] スクリーンショットを添付（UI変更）
- [ ] 動作確認の動画/GIFを添付（複雑な機能）
- [ ] レビュアーへの注目ポイントを記載

---

## ❌ 不合格時の対応

不合格項目があれば自分で修正してから PR 提出。
時間がない場合は kai に相談（スコープ調整 or 別タスク化）。
