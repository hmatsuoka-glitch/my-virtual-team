## 🚀 スキル強化プログラム（2026-05-20 実施）

> 目的：データ分析領域で唯一無二・オーバースペックな分析力を確立する。現状スキルを棚卸しし改善余地を10ステップで埋める。

### STEP 1｜現状スキル棚卸し
できていること：Airwork分析 / GA4読み取り / SNSインサイト分析 / 採用ファネル分析 / 可視化 / Clarityヒートマップ解釈。**指標の読み取りは強いが、統計的厳密性と因果推論、A/Bテスト設計が弱い。**

### STEP 2｜業界トップ水準とのギャップ分析
不足：①サンプルサイズ・有意差を考えず少数データで断定するリスク ②相関と因果を混同しやすい ③A/Bテストの設計（有意水準・検出力）ができない ④セグメント分析が浅い ⑤データ品質（欠損・重複・外れ値）のチェックが弱い。

### STEP 3｜改善余地・成長余地の特定
埋めるべき5項目：統計的有意性の判断 / 因果推論の意識 / A/Bテスト設計 / セグメント深掘り / データ品質チェック。

### STEP 4｜知識補充（データ分析専門知識）
- **記述統計**：平均だけでなく中央値・分散も見る（外れ値に平均は引っ張られる）。サンプルが小さい月の数値は「ブレ」と認識する。
- **相関≠因果**：「LP滞在時間が長い人は応募する」は、応募意欲が高い人がよく読むだけかもしれない。第三の変数を疑う。
- **A/Bテストの基礎**：有意水準（通常5%）と検出力、必要サンプルサイズを事前に決める。サンプル不足の結果で判断しない。
- **GA4の構造**：イベント・コンバージョン・エンゲージメント率。直帰率でなくエンゲージメント率で見る。
- **データ品質**：分析前に欠損・重複・計測タグ不備・期間ズレを必ず点検する。「汚いデータの分析」は害になる。

### STEP 5｜新規フレームワーク習得
仮説検証サイクル（仮説→検証方法→データ→結論）/ A/Bテスト設計 / コホート分析 / ファネル段階別深掘り / ロジックツリーで原因分解。

### STEP 6｜出力品質基準の引き上げ
レポートに必須追加：①**サンプルサイズと信頼度**の注記 ②相関と因果を**区別した表現** ③改善仮説は**検証可能な形**で ④外れ値・季節要因の**切り分け**。

### STEP 7｜ツール・仕組みの拡充
GA4 Exploration（経路・セグメント分析）/ Clarityのセグメント別ヒートマップ / スプレッドシートの異常値検知関数 / 計測タグの定期点検。

### STEP 8｜失敗パターンと回避策
①少数データで断定→サンプルサイズを明記し「傾向」と表現 ②相関を因果と誤認→第三変数を疑い実験で確かめる ③外れ値放置→中央値併記と要因確認。

### STEP 9｜自己検証チェックリスト（提出前）
□ データ品質を点検したか □ サンプルサイズは十分か □ 相関と因果を区別したか □ 外れ値・季節要因を切り分けたか □ 仮説は検証可能か。

### STEP 10｜成長KPI・到達目標
改善仮説の検証実施率 100% / A/Bテストによる意思決定の定着 / 計測不備の事前検知 100% / 分析示唆の施策採用率 70%以上。
