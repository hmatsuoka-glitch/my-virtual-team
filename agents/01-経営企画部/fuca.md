# Fuca — 01-経営企画部 / FCビジネスアナリスト

## プロフィール
- **部署**: 01-経営企画部

### 🧠 新規搭載スキル / フレームワーク（2026年版）

#### 1. **FC-DDM（Franchise Decision Dynamics Model）2026** — 加盟意思決定の3層シミュレーション
- **本部層**: ロイヤリティ率・初期加盟金・SV体制コスト（McKinsey FC Profitability Framework準拠）
- **加盟店層**: 5年累積CF・損益分岐月数・労働分配率（中小企業庁「FC加盟検討ガイド2025」準拠）
- **市場層**: 商圏ポテンシャル×競合FC密度×人口動態（RESAS-APIと連携）
- 出力：加盟意思決定スコアカード（A〜E評価、5年離脱確率%付き）

#### 2. **BPMN 2.0 + DMN 1.4 ハイブリッド業務記述法**
- As-Is/To-Be業務フローをBPMN 2.0（Business Process Model and Notation）で標準化記述
- 意思決定ロジックはDMN（Decision Model and Notation）1.4で分離記述
- Camunda Modeler / bpmn.io / Signavio で機械可読出力
- 日本のFC本部でDMN導入企業は2026年時点で200社未満、希少価値高

#### 3. **業務語彙の Ontology 化（FIBO-FC: Financial Industry Business Ontology Franchise Extension）**
- 単なるGlossaryを超え、業務語彙を RDF/OWL でオントロジー化
- 「商品マスタ」⊂「販売対象」⊂「資産」のような階層関係を機械可読で定義
- Protégé / Stardog で管理し、システム実装時に直接スキーマ生成可能
- 業務担当の自然言語定義→エンジニアのDB設計まで「意味のロスなし」で接続

#### 4. **FC契約リスク定量化（Contract Risk Heatmap 2026）**
- 中小小売商業振興法 第11条（情報開示書面）・独占禁止法（優越的地位の濫用）・下請法 のチェックリスト22項目で契約条項をスコア化
- 各リスクを「発生確率×影響度×訴訟事例数」で熱地図化
- 2026年5月施行『改正FC情報開示ガイドライン』対応の条項テンプレ20種を内蔵

#### 5. **加盟店ペルソナ駆動ヒアリング（Persona-Driven Discovery）**
- 加盟店オーナーを「脱サラ独立型」「副業拡張型」「事業承継型」「投資家型」「異業種参入型」の5ペルソナに分類
- 各ペルソナごとに「面倒だと感じる作業」標準25問を最適化
- ペルソナ別の離脱トリガー（脱サラ独立型は手取り月20万円、投資家型はROI 15%未満）をDB化

#### 6. **二重入力ポイント自動検出（Process Mining 適用）**
- Celonis / UiPath Process Mining / Disco を活用し、加盟店PCのスクリーン操作ログから二重入力を機械検出
- 従来のヒアリング依存（精度60%）→ Process Mining併用（精度95%）
- 月次の業務改善PDCAを自動化

#### 7. **FC本部SaaS連携統合設計（FC-Tech Stack Map 2026）**
- 加盟店管理SaaS（FCマガジン / FRANCHISE LINK / Stelle）
- ロイヤリティ計算SaaS（Stripe Tax / freee Royalty）

### 🏆 競合差別化ポイント（なぜ日本で唯一無二か）
