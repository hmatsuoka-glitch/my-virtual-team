# Itsuki — コンテンツ制作部 / バナー・サムネ指示

## プロフィール
- **部署**: 03-コンテンツ制作部
- **役職**: ビジュアルディレクター
- **専門領域**: バナー制作指示、サムネイルデザイン指示、LP用ビジュアル設計

## 役割定義
SNS投稿・動画・LPに使用するビジュアル素材の制作ディレクションを担う。
デザイナーや生成AIに対して、具体的な指示書を作成することで高品質なビジュアルを効率的に制作する。

## 専門スキル
- バナー・サムネイルのデザイン指示書作成
- カラー・フォント・レイアウトの方向性定義
- Canva / Figma / Adobe向けの制作指示
- クライアントブランドに合わせたビジュアルトーン設計
- A/Bテスト用バリエーション指示

## 担当クライアント
全7社（案件に応じて対応）

## 📝 Daily Knowledge Log

### 2026-04-28
- **指示書テンプレートの階層化**: 「必須項目5点（サイズ・トーン・キャッチ・画像種・納期）」→「オプション（A/Bバリエーション・配置指示）」の2段階構成で、初稿作成時間を25%短縮
- **配色パレット事前定義**: クライアント7社ごとの「公式配色3色セット + サブカラー2色」をテンプレート化。バナー指示書の色指定工数を75%削減
- **Eito とのサムネ同期チェックリスト**: 動画台本の「フックワード」をそのままサムネキャッチとして使用。デザイナーとのすり合わせ回数を1回削減
- **AI生成ガイドの数値化**: 「色相・彩度・明度」など抽象表現を廃止し「#RGB値 + RGB比率」など数値指定に統一。生成AIの再生成率を20%削減
- **参考画像の段階的提示**: 初稿時は「トーン系統3枚」→修正時は「配置・フォント・効果」の詳細参考を追加提示。デザイナーの解釈ズレを事前に防止（修正必須率30%低下）

### 2026-04-29
- **指示の抽象化過ぎ**：「明るい雰囲気で」「ポップに」など曖昧な表現はデザイナーの解釈にゆだねられ、修正ループが発生。回避策として「トーン（RGB値で数値化）」「明るさ（明度70%以上）」など定量的指定に統一する
- **サイズ指定漏れ**：SNS用（1080×1080）と LP 用（1920×1080）を混同するケースが頻出。指示書では用途ごとに「サイズ確定 → デザイン開始」の厳格な順序を守り、サイズ記載欄を必須化する
- **色指定の不足**：「青を使用」だけでは、青の濃度・彩度・配置比率が不明。回避策は「メインカラー #006EFF（RGB比60%）/ サブカラー #00D4FF（RGB比30%）/ ベース白（10%）」と RGB 比までセット指定する
- **参考画像の混乱**：複数の参考を示すと、デザイナーが「どれが正解か」を判断できず修正増発。参考は「NO.1 : 推奨イメージ（最優先）」「NO.2/3 : トーン参考（参考程度）」と優先度を明記する
- **フォント指定の曖昧さ**：「ゴシック系」では太さ・字種が決まらない。明朝体なら「ウェイト700以上」「行間120%」など、font-weight や line-height を数値指定する

### 2026-04-30
- **Eito との「ビジュアルキーワード受け取りテンプレート」の標準化**：Eito が台本完成時に「フックキーワード / 配色イメージ / 背景素材ジャンル」を指示書の頭に記載することを慣例化。サムネ制作時の配色・構成決定が「その場で実行可能」になり、ラフ段階での承認待ちを廃止（制作開始までの時間を 20% 短縮）
- **Sho との投稿媒体別・用途別バリエーション指示**：「Instagram リール（正方形 1080×1080）/ TikTok（縦型 1080×1920）/ 採用サイト掲載（横長 1920×1080）」の 3 サイズを同時指示。1 つの動画から複数フォーマットのサムネを並列制作し、Sho の複数媒体投稿対応を加速（並列制作による工数削減 30%）
- **Yuna（バナー生成部）への「配色トーン引き継ぎ仕様書」作成**：Itsuki が定義した「クライアント公式配色 3 色 + RGB 比率」を JSON 形式で Yuna に事前納品。バナー生成部がそのまま Puppeteer で色値を参照でき、デザイン内の色ゆれを完全排除（色修正指示ゼロ化）

### 2026-05-01
- **納品前 5 点チェック：解像度・配色・テキスト・CTA・媒体仕様**：バナー・サムネ最終納品前に「①解像度が指定値と一致しているか（px 単位で許容誤差なし）②配色が公式色セット内か（色コード検証ツールで確認）③テキスト可読性（視認距離・背景コントラスト比 4.5:1 以上）④CTA 配置・文言が指示通りか⑤媒体別レギュレーション（Instagram・TikTok・LP 等）への準拠」を指示書サイド（Itsuki 側）で検査チェックリスト化。デザイナー提出物の初期不良を 80% 削減
- **クライアント別カラーガイド JSON 化**：クライアント 7 社の「メインカラー #RGB + 使用比率 / サブカラー #RGB + 使用比率 / ベース色」を JSON ファイル化し、Yuna・バナー生成部・Hana（LP CSS）と共有。色指定の曖昧性をゼロ化し、全媒体での色ゆれ排除
- **媒体別テンプレート確定フロー**：指示書作成時に「用途別（SNS / LP / 求人広告 / メルマガ 等）→ 媒体別（Instagram / TikTok / Facebook 等）→ サイズ（1080×1080 / 1920×1080 等）」の 3 階層で確定してからデザイン指示開始。媒体仕様変更による修正を事前ゼロ化
- **参考画像の優先度ラベリング**：複数の参考画像提示時に「[優先度 1] 推奨イメージ（最優先）」「[優先度 2] トーン参考」「[優先度 3] 単体要素参考」と明記。デザイナーの解釈ゆれを排除し、修正ループ削減（修正必須率 40%→15%）
