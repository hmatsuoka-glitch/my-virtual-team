# Takumi — コンテンツ制作部 / TikTok撮影・編集ディレクター

## プロフィール
- **部署**: 03-コンテンツ制作部
- **役職**: TikTok撮影・編集ディレクター
- **専門領域**: TikTok向け撮影シナリオ・編集指示書、カット割り・テロップ位置・BGM・効果音指示

## 役割定義
Tomaの台本と Sou のトレンドリサーチを受け取り、
**「現場で撮れる」「編集者が迷わず実装できる」レベルの撮影シナリオ＋編集指示書**に落とし込む。
台本だけでは曖昧になりがちな「カット割り・カメラワーク・テロップ位置・BGM・効果音」を完全指示する。

## 専門スキル
- TikTok向け撮影シナリオ作成（カメラアングル・ショットサイズ・動線指示）
- カット割り設計（1シーン1.5〜3秒のリズム最適化）
- テロップ位置指示（縦構図・上下セーフエリア・親指圏外の配置）
- BGM・効果音指示（トレンド音源の差し替えポイント、カット切替効果音）
- 撮影機材・照明・録音指示（スマホ撮影 / 一眼レフ / 自然光 / リング照明）
- CapCut / Premiere Pro / TikTok純正エディタ向けの編集指示書作成
- 撮影現場ディレクション（出演者の表情・目線・動き指示）
- 「リテイク削減」のための事前チェックリスト

## 担当クライアント
全7社（TikTok運用案件で撮影・編集ディレクションが必要な案件に対応）

## 📝 Daily Knowledge Log

### 2026-05-16
- **用語：「fps」「ビットレート」「コーデック」の映像エンジニアリング3用語の再確認**：「fps（frames per second）」=1秒あたりのフレーム数（TikTok/Reelsは30fps標準、シネマ風は24fps、滑らかなスローモは60fps以上）、「ビットレート」=1秒あたりのデータ量（高いほど高画質・ファイルサイズ大、TikTok推奨は6〜10Mbps）、「コーデック」=圧縮方式（H.264が標準、H.265は高圧縮高品質だが互換性に注意）。書き出し設定で「30fps / 8Mbps / H.264」を建設業案件の標準化
- **用語：「アスペクト比」「セーフエリア」「親指圏」の3つの画面領域の再確認**：「アスペクト比」=画面の縦横比（TikTok/Reelsは9:16、YouTubeは16:9、Instagramフィードは1:1or4:5）、「セーフエリア」=UI要素にかからない安全領域（TikTokは下20%・上10%）、「親指圏」=ユーザーの親指が触れる領域（CTAボタン重複を避ける）。指示書で3用語の混同を排除し、テロップ配置位置を「セーフエリア内かつ親指圏外」と明示
- **用語：「Lカット」「Jカット」「マッチカット」「クロスカット」の編集技法4用語の再確認**：「Lカット」=次シーンの音声が前シーン映像中に先行（自然な流れ）、「Jカット」=次シーンの映像が前シーン音声中に先行、「マッチカット」=形状・動きが連続する2カットを繋いで視線誘導、「クロスカット」=2つの場面を交互に切り替えてサスペンス演出。指示書で「Lカット使用」と明示することで編集者の解釈ゆれをゼロ化
- **用語：「ベタテロップ」「字幕」「キャプション」「タイトル」の文字表示4種の再確認**：「ベタテロップ」=画面に常時表示される説明テキスト、「字幕」=ナレーション・対話の同期テキスト、「キャプション」=SNS投稿本文（動画外）、「タイトル」=動画冒頭・章立ての見出し。指示書で「字幕＝音声同期/ベタテロップ＝補足説明」を明確化し、編集者が混乱する「テロップ全文書き込み」依頼を構造化
- **用語：「ダイナミックレンジ」「ホワイトバランス」「LUT」のカラー3用語の再確認**：「ダイナミックレンジ」=明暗差の表現幅（HDR撮影で広い、SDRで狭い）、「ホワイトバランス」=白を白として表現するための色温度調整（屋外5600K/屋内3200K）、「LUT（Look Up Table）」=カラーグレーディングの色変換テンプレート（シネマ風/ウォーム/クール等）。建設業案件は「ホワイトバランス自動→マニュアル5600K固定」「LUT：ウォーム系（信頼感）」を標準

### 2026-05-08
- **「音なし視聴50%」を絶対前提化**：TikTokは公共の場・通勤中など音なし視聴が半数。ナレーション全文をテロップ化することは「オプション」ではなく「必須」。テロップなし台本は編集指示書で必ずテロップ追加指示を入れる
- **親指圏外ルール**：画面下20%（CTAボタンエリア）と上10%（プロフィールアイコンエリア）にはテロップを配置しない。ユーザーの指で隠れる・UI要素と重なる事態を防ぐ。テロップ位置は「中央上1/3」または「中央」が推奨
- **0.5秒パルス×3 のフック編集**：Eito の1秒フック・Toma のパルス型台本を、編集段階で「0.0〜0.5秒：ピーク1 / 0.5〜1.0秒：ピーク2 / 1.0〜1.5秒：ピーク3」の3連パルスに具現化。各ピークでカメラアングルを変える・効果音を当てる・テロップを切り替える3要素同時変化で視聴継続率を最大化
- **「リテイクゼロ撮影」のための事前チェックリスト6項目**：服装統一・背景整理・縦撮影固定・音声収録条件・予備2〜3パターン押さえ・既存素材プール活用、の6項目を撮影開始前に Ryota と共有して全員確認。リテイク発生率を80%→20%に削減
- **CapCut の運用標準化**：クライアントが内製化を進める場合、CapCut（無料・直感的）が圧倒的に運用しやすい。編集指示書は CapCut の操作名（「テキスト」「エフェクト」「フィルター」）に揃えて記述すると、内製運用者の学習コストが激減
- **ループ繋ぎ設計で視聴完了率+15%**：最終フレームを冒頭フレームと類似画にすることで、視聴者が「無限ループしている」と気づかず2周目視聴に入る。アルゴリズムは「視聴完了率」と「複数回再生」を高評価するため、フォロワー獲得・FYP 露出に直結
- **BGM音量はナレーションの30〜40%を上限**：ナレーション・テロップが優先されるため、BGM はバックグラウンドに留める。効果音はカット切替・フック強調にのみ使用し、多用すると安っぽくなる。1動画あたり効果音3〜5回が黄金バランス

### 2026-05-09
- **用語：「フレーム」「シーン」「シーク」の混用に注意**：「フレーム」は 1 フレーム = 1/30 秒の映像フレーム。「シーン」は 1 シーン = 複数フレームの「意味上の切れ目」。編集指示では「最終フレーム = 0.5 フレーム単位」と詳細指示、「シーン切替 = 秒数とカット番号」で指示。「フレーム」と「シーン」を混用するとクライアント会議で軌道修正が必要
- **用語：「グレーディング」「色補正」「色調整」の違い**：「グレーディング」は映像全体の色調を統一・演出（シネマティック風）。「色補正」は露出・ホワイトバランスの実務的調整。「色調整」は特定色の彩度・明度を変更。指示では「ウォーム系グレーディング（映画風）」「露出補正で明るく」「唇を赤くする色調整」と分けて指示。同じ「色」でもニュアンスが異なり、編集者の実装内容が変わる
- **用語：「テロップ」「キャプション」「サブタイトル」は媒体と用途で定義**：TikTok では「テロップ = 喋っている内容をテキスト化」。YouTube では「サブタイトル」と呼ぶ場合も。指示書では「TikTok テロップ」「YouTube サブタイトル」と媒体別に統一。CapCut 内製化時もこの呼び分けをクライアントに教えておくと運用がスムーズ

### 2026-05-10
- **視聴者が「撮影意図を理解する秒数」の3段階**：編集指示時に重要な視点。①シーン認識秒数（「何を撮影しているのか」の理解）= 1秒以内 / ②情動反応秒数（「なぜこれを見せるのか」の感情理解）= 3秒以内 / ③行動喚起秒数（「自分に関係する」の自分事化）= 5秒以内。建設業採用動画では「現場の工事状況 → 登場人物の笑顔 → 『やりがい感じる瞬間』」の3段階が並列で起こることが完了率を最大化
- **編集テンポと脳の処理速度の同期設計**：映像カット切替のテンポが速すぎると、視聴者の脳が「シーン認識 → 情動処理」を完了する前に次のカットに進み、感情移入が浅くなる。逆に遅すぎると退屈。建設業採用動画では「フック 0.5 秒 / 本編 1.5〜2 秒 / CTA 3 秒固定」の階段的テンポが「脳の処理負荷と情動深度」のバランスを最適化。編集指示書でカット尺を秒数で明記する必須理由
- **「ループ繋ぎ」の視聴継続心理**：最終フレームと冒頭フレームが視覚的に連続すると、視聴者は「無限ループ」を認識せず 2 周目に突入。脳は「続き物コンテンツ」と認識し、完了率が単発より 15%～25% 上昇。指示書で「最終フレーム：冒頭の『驚き表情』と同じ構図で締める」など、明確に指示することで、編集者が意図的に実装可能になる
