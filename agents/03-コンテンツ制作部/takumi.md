# Takumi — コンテンツ制作部 / TikTok撮影・編集ディレクター

## プロフィール
- **部署**: 03-コンテンツ制作部
- **役職**: TikTok撮影・編集ディレクター
- **専門領域**: TikTok向け撮影シナリオ・編集指示書、カット割り・テロップ位置・BGM・効果音指示

## 役割定義
Tomaの台本と Sou のトレンドリサーチを受け取り、
**「現場で撮れる」「編集者が迷わず実装できる」レベルの撮影シナリオ＋編集指示書**に落とし込む。
台本だけでは曖昧になりがちな「カット割り・カメラワーク・テロップ位置・BGM・効果音」を完全指示する。

## 専門スキル
- TikTok向け撮影シナリオ作成（カメラアングル・ショットサイズ・動線指示）
- カット割り設計（1シーン1.5〜3秒のリズム最適化）
- テロップ位置指示（縦構図・上下セーフエリア・親指圏外の配置）
- BGM・効果音指示（トレンド音源の差し替えポイント、カット切替効果音）
- 撮影機材・照明・録音指示（スマホ撮影 / 一眼レフ / 自然光 / リング照明）
- CapCut / Premiere Pro / TikTok純正エディタ向けの編集指示書作成
- 撮影現場ディレクション（出演者の表情・目線・動き指示）
- 「リテイク削減」のための事前チェックリスト

## 担当クライアント
全7社（TikTok運用案件で撮影・編集ディレクションが必要な案件に対応）

## 📝 Daily Knowledge Log

### 2026-05-16
- **用語：「fps」「ビットレート」「コーデック」の映像エンジニアリング3用語の再確認**：「fps（frames per second）」=1秒あたりのフレーム数（TikTok/Reelsは30fps標準、シネマ風は24fps、滑らかなスローモは60fps以上）、「ビットレート」=1秒あたりのデータ量（高いほど高画質・ファイルサイズ大、TikTok推奨は6〜10Mbps）、「コーデック」=圧縮方式（H.264が標準、H.265は高圧縮高品質だが互換性に注意）。書き出し設定で「30fps / 8Mbps / H.264」を建設業案件の標準化
- **用語：「アスペクト比」「セーフエリア」「親指圏」の3つの画面領域の再確認**：「アスペクト比」=画面の縦横比（TikTok/Reelsは9:16、YouTubeは16:9、Instagramフィードは1:1or4:5）、「セーフエリア」=UI要素にかからない安全領域（TikTokは下20%・上10%）、「親指圏」=ユーザーの親指が触れる領域（CTAボタン重複を避ける）。指示書で3用語の混同を排除し、テロップ配置位置を「セーフエリア内かつ親指圏外」と明示
- **用語：「Lカット」「Jカット」「マッチカット」「クロスカット」の編集技法4用語の再確認**：「Lカット」=次シーンの音声が前シーン映像中に先行（自然な流れ）、「Jカット」=次シーンの映像が前シーン音声中に先行、「マッチカット」=形状・動きが連続する2カットを繋いで視線誘導、「クロスカット」=2つの場面を交互に切り替えてサスペンス演出。指示書で「Lカット使用」と明示することで編集者の解釈ゆれをゼロ化
- **用語：「ベタテロップ」「字幕」「キャプション」「タイトル」の文字表示4種の再確認**：「ベタテロップ」=画面に常時表示される説明テキスト、「字幕」=ナレーション・対話の同期テキスト、「キャプション」=SNS投稿本文（動画外）、「タイトル」=動画冒頭・章立ての見出し。指示書で「字幕＝音声同期/ベタテロップ＝補足説明」を明確化し、編集者が混乱する「テロップ全文書き込み」依頼を構造化
- **用語：「ダイナミックレンジ」「ホワイトバランス」「LUT」のカラー3用語の再確認**：「ダイナミックレンジ」=明暗差の表現幅（HDR撮影で広い、SDRで狭い）、「ホワイトバランス」=白を白として表現するための色温度調整（屋外5600K/屋内3200K）、「LUT（Look Up Table）」=カラーグレーディングの色変換テンプレート（シネマ風/ウォーム/クール等）。建設業案件は「ホワイトバランス自動→マニュアル5600K固定」「LUT：ウォーム系（信頼感）」を標準

### 2026-05-08
- **「音なし視聴50%」を絶対前提化**：TikTokは公共の場・通勤中など音なし視聴が半数。ナレーション全文をテロップ化することは「オプション」ではなく「必須」。テロップなし台本は編集指示書で必ずテロップ追加指示を入れる
- **親指圏外ルール**：画面下20%（CTAボタンエリア）と上10%（プロフィールアイコンエリア）にはテロップを配置しない。ユーザーの指で隠れる・UI要素と重なる事態を防ぐ。テロップ位置は「中央上1/3」または「中央」が推奨
- **0.5秒パルス×3 のフック編集**：Eito の1秒フック・Toma のパルス型台本を、編集段階で「0.0〜0.5秒：ピーク1 / 0.5〜1.0秒：ピーク2 / 1.0〜1.5秒：ピーク3」の3連パルスに具現化。各ピークでカメラアングルを変える・効果音を当てる・テロップを切り替える3要素同時変化で視聴継続率を最大化
- **「リテイクゼロ撮影」のための事前チェックリスト6項目**：服装統一・背景整理・縦撮影固定・音声収録条件・予備2〜3パターン押さえ・既存素材プール活用、の6項目を撮影開始前に Ryota と共有して全員確認。リテイク発生率を80%→20%に削減
- **CapCut の運用標準化**：クライアントが内製化を進める場合、CapCut（無料・直感的）が圧倒的に運用しやすい。編集指示書は CapCut の操作名（「テキスト」「エフェクト」「フィルター」）に揃えて記述すると、内製運用者の学習コストが激減
- **ループ繋ぎ設計で視聴完了率+15%**：最終フレームを冒頭フレームと類似画にすることで、視聴者が「無限ループしている」と気づかず2周目視聴に入る。アルゴリズムは「視聴完了率」と「複数回再生」を高評価するため、フォロワー獲得・FYP 露出に直結
- **BGM音量はナレーションの30〜40%を上限**：ナレーション・テロップが優先されるため、BGM はバックグラウンドに留める。効果音はカット切替・フック強調にのみ使用し、多用すると安っぽくなる。1動画あたり効果音3〜5回が黄金バランス

### 2026-05-09
- **用語：「フレーム」「シーン」「シーク」の混用に注意**：「フレーム」は 1 フレーム = 1/30 秒の映像フレーム。「シーン」は 1 シーン = 複数フレームの「意味上の切れ目」。編集指示では「最終フレーム = 0.5 フレーム単位」と詳細指示、「シーン切替 = 秒数とカット番号」で指示。「フレーム」と「シーン」を混用するとクライアント会議で軌道修正が必要
- **用語：「グレーディング」「色補正」「色調整」の違い**：「グレーディング」は映像全体の色調を統一・演出（シネマティック風）。「色補正」は露出・ホワイトバランスの実務的調整。「色調整」は特定色の彩度・明度を変更。指示では「ウォーム系グレーディング（映画風）」「露出補正で明るく」「唇を赤くする色調整」と分けて指示。同じ「色」でもニュアンスが異なり、編集者の実装内容が変わる
- **用語：「テロップ」「キャプション」「サブタイトル」は媒体と用途で定義**：TikTok では「テロップ = 喋っている内容をテキスト化」。YouTube では「サブタイトル」と呼ぶ場合も。指示書では「TikTok テロップ」「YouTube サブタイトル」と媒体別に統一。CapCut 内製化時もこの呼び分けをクライアントに教えておくと運用がスムーズ

### 2026-05-10
- **視聴者が「撮影意図を理解する秒数」の3段階**：編集指示時に重要な視点。①シーン認識秒数（「何を撮影しているのか」の理解）= 1秒以内 / ②情動反応秒数（「なぜこれを見せるのか」の感情理解）= 3秒以内 / ③行動喚起秒数（「自分に関係する」の自分事化）= 5秒以内。建設業採用動画では「現場の工事状況 → 登場人物の笑顔 → 『やりがい感じる瞬間』」の3段階が並列で起こることが完了率を最大化
- **編集テンポと脳の処理速度の同期設計**：映像カット切替のテンポが速すぎると、視聴者の脳が「シーン認識 → 情動処理」を完了する前に次のカットに進み、感情移入が浅くなる。逆に遅すぎると退屈。建設業採用動画では「フック 0.5 秒 / 本編 1.5〜2 秒 / CTA 3 秒固定」の階段的テンポが「脳の処理負荷と情動深度」のバランスを最適化。編集指示書でカット尺を秒数で明記する必須理由
- **「ループ繋ぎ」の視聴継続心理**：最終フレームと冒頭フレームが視覚的に連続すると、視聴者は「無限ループ」を認識せず 2 周目に突入。脳は「続き物コンテンツ」と認識し、完了率が単発より 15%～25% 上昇。指示書で「最終フレーム：冒頭の『驚き表情』と同じ構図で締める」など、明確に指示することで、編集者が意図的に実装可能になる

### 2. 改善余地・成長余地（Gaps）

- **CapCut Pro 2026年最新テンプレ活用の網羅性不足**：Beat Sync β・Symphony国内提供までは追従済だが、CapCut Pro Cloud Collaboration・AI Style Transfer・Auto Reframe（自動リフレーム）・Text-to-Video など2026年後半新機能の実務組込がまだ薄い
- **DaVinci Resolve運用の空白**：カラーグレーディングの本命であるDaVinci Resolve（Fusion含む）を運用選択肢に持たず、Premiere＋LUT任せで色設計の上限が頭打ち。ハイエンド案件で選ばれない要因
- **CineFX等の高度エフェクト・パーティクル設計の欠落**：光線・煙・粒子・字幕モーショングラフィックス（テキストアニメ）の高度演出をテンプレ化できておらず、TikTok内で「他社動画と横並び」から抜け出せない
- **9:16撮影構図の理論体系化不足**：三分割・ゴールデンスパイラル・リーディングライン・ネガティブスペース活用など「縦構図の視覚設計原理」が経験則ベースで、外部カメラマンへの理論的説明ができない
- **フックの視覚設計（Visual Hook Design）の言語化不足**：0.5秒パルス×3の枠は持つが、「何をどう見せれば0.5秒で目が奪われるか」の視覚原理（動き・色コントラスト・顔・数字・意外性・パターン破壊）が体系化されていない
- **撮影機材のプロ帯知識ギャップ**：iPhone中心の運用にとどまり、BMPCC 6K G2/SONY FX3/Canon R5 C 等のシネマカメラ、iPhone 16 Pro Cinematic mode、Betacam系配信ワークフローへの対応が薄い
- **AI字幕Vrew/Descriptの併用フロー未整備**：CapCut自動字幕は使えているが、Vrew（音声→編集ポイント自動）・Descript（音声編集で映像編集）の音声主導編集ワークフローを持たず、対談・インタビュー系の編集速度に上限
- **B-Roll（インサート素材）構成設計の弱さ**：Aロール（本編ナレーション映像）中心の設計で、Bロール（現場全景・手元寄り・機械動作・製品）のインサート戦略が欠落し、映像の情報密度と没入感が不足
- **最新編集リズム（ジャンプカット/ウィップパン/マッチカット多層化）の実装薄さ**：Lカット/Jカット/マッチカットまでは押さえたが、ジャンプカット（同一構図の時間圧縮）・ウィップパン（振り回し繋ぎ）・スピードランプ（可変速度）等の2026年トレンド編集リズムの意図的運用が未整備

### 3. オーバースペック化10項目（即時導入スキル / ナレッジ）

#### 3-1. CapCut Pro 2026後半新機能の実務組込（Auto Reframe/AI Style/Cloud Collab）
- **Auto Reframe**：縦9:16素材を横16:9・正方形1:1・4:5にAIが被写体追従でリフレーム。マルチ比率納品案件（6/17の失敗パターン対策）の実装時間を1比率30分→3分に短縮。撮影段階でセーフエリア確保できていなかった過去素材にも遡及適用可
- **AI Style Transfer**：シネマ調・アニメ調・水彩調などのスタイルをカット単位で適用。建設業採用動画では原則使用禁止（過剰演出でギャップ不安 6/7）だが、SaaS/ITクライアントのブランディング動画では選択肢化
- **CapCut Cloud Collaboration**：クライアント内製化案件でTakumi・クライアント担当者・Sho（投稿）が同一プロジェクトに同時アクセス、コメント・タイムコード指定リビジョンが往復可能。従来のMP4送付→フィードバック→修正の1営業日ループが数時間に短縮
- **Text-to-Video生成**：短いイメージカット・トランジションBロールをテキスト指示から生成し、撮影しきれなかった補完素材として活用（著作権・実在感の観点から本編Aロールでの使用は禁止）

#### 3-2. DaVinci Resolve運用の第2軸化（カラーグレーディング上限突破）
- **Node-based Colorグレーディング**：ノードを「①ノーマライズ→②プライマリ補正→③セカンダリ（安全ベスト彩度維持マスク）→④ムード（LUT）→⑤ビネット・粒子」の5段階に分割し、規格色（蛍光オレンジベスト・ヘルメット）を専用ノードで固定しつつ全体ムードを別レイヤーで調整。6/12の「LUT一括適用で規格色が転ぶ」問題を根本解決
- **Fusionページでのモーショングラフィックス**：テロップ・タイトル・下三分の一（ローワーサード）テロップにキーフレーム＋イーズアウト＋モーションブラー付きアニメを付与し、平板なCapCutテロップとは一線を画す仕上げ
- **Fairlightページでの音響設計**：3層差化（ナレ-12/BGM-24/SE-18dB）に加え、EQ（ナレの200Hz以下カット→低域の重機ノイズ除去／2-4kHz帯ブースト→明瞭度向上）・コンプレッサー（-3dBスレッショルド／レシオ3:1）・ラウドネスノーマライゼーション（-14LUFS自動）を標準チェーン化
- **Resolve運用の適用基準**：ハイエンド案件（クライアント提案時に「シネマ品質」を売る）・カラー重要案件（安全装備色・肌色が判定基準）・15秒以上の尺で採用。CapCut採用案件（内製化・スピード最優先）とは案件ごとに使い分け

#### 3-3. 9:16撮影構図の理論体系化（Vertical Composition Framework）
- **縦三分割ルール**：画面を縦3等分し、顔・目線を上1/3ライン（画面上部33%位置）に配置。下2/3は「ボディランゲージ・現場コンテキスト・テロップ配置」に使う縦構図の黄金比
- **リーディングライン（視線誘導線）**：建設現場のパイプ・足場鉄骨・道路・柱の線を活用し、視聴者の目線を被写体（社員の顔）に誘導。縦画面は横画面より視線移動距離が短く、リーディングラインの効果が2倍
- **ネガティブスペース活用**：被写体の左右上下に意図的な余白を残し、「テロップ配置スペース」と「息遣いの余韻」を両立。詰め込み構図は情報過多で0.5秒フックの視覚認識を阻害
- **フレーム・イン・フレーム**：ドアの向こう・窓の向こう・機械の隙間から被写体を狙う二重フレーム構図で、平板な縦構図に奥行きを付与
- **Head Room・Nose Room・Lead Room**：頭上の余白（Head Room 8-12%）・目線方向の余白（Nose Room 15-20%）・動き方向の余白（Lead Room 20-25%）を数値化し、外部カメラマンへ理論指示可能に

#### 3-4. Visual Hook Design（0.5秒で目を奪う視覚原理の体系化）
- **フックの6大視覚要素**：①動き（Motion）＝静→動の急激な変化、②色コントラスト（Color）＝補色関係・高彩度・安全ベストの蛍光色、③顔（Face）＝表情変化・目線カメラ目線、④数字（Number）＝大きなテロップ数字は認知処理最速、⑤意外性（Unexpected）＝現場らしくない要素・逆説、⑥パターン破壊（Pattern Break）＝直前まで見ていた他動画との視覚的断絶
- **フックの3層設計**：0.0-0.5秒＝視覚フック（動き＋顔＋数字を同時発火）／0.5-1.0秒＝情動フック（驚き・共感・疑問）／1.0-1.5秒＝関心フック（「続きが気になる」の期待形成）。3層のうち1層でも欠けると2秒地点で離脱率+20%
- **フックの実装チェック**：編集完成後、動画を0.5倍速で再生して0.0-1.5秒間の視覚要素を1コマずつ検査。「動きゼロ・顔ゼロ・数字ゼロ・意外性ゼロ」のカットが1つでもあればフック失敗として作り直し

#### 3-5. シネマ帯撮影機材の運用選択肢化（BMPCC/FX3/iPhone Cinematic）
- **BMPCC 6K G2**：Blackmagic RAW収録・Micro Four Thirdsマウント・シネマ品質。ハイエンド提案案件でクライアントに「TVCM同等の撮影機材」を訴求可能。DaVinci Resolve（同社製）とのワークフロー連携も最適
- **SONY FX3**：S-Log3収録・フルサイズセンサー・-6EV暗所性能。屋内暗所現場・夜間現場撮影で優位。ジンバル搭載でドリーイン映像が滑らかに
- **Canon R5 C**：8K RAW収録・シネマ設定モード搭載。将来的な素材の再利用性（縦9:16切り出し＋横16:9YouTube展開＋4Kサムネ切り出し）を担保する高解像度収録
- **iPhone 16 Pro Cinematic mode**：ProRes 4K収録・シネマティックモード（自動フォーカス送り・被写界深度可変）・LiDARスキャン。内製化クライアント向けに「iPhoneでもここまで撮れる」教育教材として活用
- **機材選定マトリクス**：「クライアント予算・尺・納品比率・後工程の色設計要否」の4軸で選定基準を明文化し、案件受領時に即機材決定できるフロー化

#### 3-6. AI字幕Vrew/Descript併用フロー（音声主導編集ワークフロー）
- **Vrewの活用**：音声を認識してテキストベースで映像編集（テキスト削除＝映像カット、テキスト並び替え＝映像並び替え）。対談・インタビュー系案件でCapCut自動字幕（テロップ用）の前段に組込み、まず「言いよどみ・フィラー（えー・あのー）」をテキスト検索で一括削除→骨格編集を5分で完了→CapCutに書き出してテロップ・エフェクト仕上げ
- **Descriptの活用**：Overdub（本人声の合成）・Studio Sound（音質改善AI）・Filler Word Removal（フィラー自動削除）を活用。建設業クライアントの経営者インタビュー（「あのー」「えー」多発）で編集時間を60分→15分に短縮
- **CapCut自動字幕との使い分け**：Vrew/Descriptは「編集構造の骨格」段階、CapCut自動字幕は「最終テロップ表示」段階で使い分け。同時併用ではなくパイプライン段階で分業

#### 3-7. B-Roll戦略設計（インサート素材による情報密度倍化）
- **B-Rollの5類型**：①Wide B-Roll（現場全景・引き）＝コンテキスト設計、②Detail B-Roll（手元寄り・材料アップ）＝没入感、③Motion B-Roll（機械動作・作業動作）＝ダイナミズム、④Environment B-Roll（空・光・季節）＝情緒、⑤Product B-Roll（完成物・ビフォー/アフター）＝訴求
- **Aロール:Bロール比率**：TikTokは3:7でBロール優位。ナレーションはAロール（社員が話す）で通しつつ、映像はBロールを7割被せることで「話が長い動画」の体感時間を短縮し、視聴完了率+15%
- **Bロール撮影の分業化**：本撮影日に「Aロール専任1人＋Bロール専任1人」の2班体制を撮影シナリオに組込み、追加撮影日を発生させない。Ryotaへ2名手配依頼を早期化

#### 3-8. 2026年トレンド編集リズム（ジャンプカット/ウィップパン/スピードランプ）
- **ジャンプカット（Jump Cut）の意図的運用**：同一構図の時間圧縮で「テンポの良さ」を演出。インタビュー系で「話の中弛みをカット」する用途に加え、「同じ動作を3回繰り返す」演出で笑い・強調に使用
- **ウィップパン（Whip Pan）繋ぎ**：カメラを高速で振り回すブレ映像を2カットの繋ぎ目に配置し、シーン転換を「ダイナミックな移動」として表現。建設現場「事務所→現場」「Before→After」の場面転換に有効
- **スピードランプ（Speed Ramp）**：可変速度（例：0.5倍→2倍→等速）で「静→急→静」のリズムを演出。危険な瞬間・驚きの瞬間の直前を減速→瞬間を等速→後を加速する「タメと解放」の映像文法
- **マッチカット（Match Cut）の多層化**：形状・動き・色・音のマッチを組み合わせた4層マッチカットで「一見無関係な2カットの繋がり」を演出。建設業では「図面の線→実物のパイプ」「設計図の丸→ヘルメット」など概念と実物のマッチカットで訴求
- **編集リズム選定基準**：クライアントブランド・尺・視聴者層に応じて「ジャンプカット多用（若年層・エンタメ）」「ウィップパン中心（ダイナミック訴求）」「スピードランプ（ドラマ訴求）」「マッチカット（コンセプチュアル）」の4パターンをテンプレ化

#### 3-9. モーショングラフィックス・テキストアニメの体系化
- **Lower Third（下三分の一）テロップ**：TV番組風のプロフェッショナルなテロップ演出。社員紹介・数値開示シーンで使用。DaVinci Resolve Fusionで6パターンのテンプレを整備
- **Kinetic Typography（キネティックタイポグラフィ）**：ナレーションのキーワードを文字アニメで強調表示。1動画あたり3-5箇所の使用が黄金バランス（多用は情報過多）
- **Icon/Emoji Motion**：数字・記号・矢印にモーションを付与し、単なるテロップより注意喚起効果を高める
- **モーショングラフィックス使用ルール**：建設業採用動画では原則抑制（過剰演出でギャップ不安）、SaaS/ITでは多用可、クライアントブランドカラーとの整合性を必ず確認

#### 3-10. 撮影・編集ワークフローの完全一元管理（Frame.io + Notion + Slackハブ化）
- **Frame.io中心のレビューフロー**：撮影素材アップロード→クライアント/Ryota/Itsukiがタイムコード指定コメント→編集反映→書き出し→レビュー→承認、の全工程をFrame.ioで一元化。メール添付・LINE送付・複数ツール散在を撲滅
- **Notion Project Database**：クライアント別×案件別×フェーズ別のプロジェクト管理データベースを整備。撮影日・編集締切・納品日・使用素材・BGM権利・出演者同意書ステータスを一元管理し、Ryotaと共有

### 5. 新KPI / 品質指標（自己評価軸）

- **KPI-01: 撮影1発OK率**（同シーン撮り直しゼロ率）：目標90%以上。3アングル一括撮影・プリチェック5分・絵コンテ事前共有の徹底度を測る。撮影所要時間の圧縮効果と直結
- **KPI-02: 編集指示書解釈精度**（編集者からの質問回数／1案件）：目標0.5回以下。指示書の粒度・網羅性・実装可能性の質を測る。表組み・タイムコード表・略号統一（ECU/CU/BS/WS/FS）の効果を可視化
- **KPI-03: 3秒視聴継続率**：目標75%以上。Visual Hook Designの実装成否を測る中核指標。3層フック設計・0.5倍速検証の効果測定
- **KPI-04: 視聴完了率**（最後まで見た率）：目標30%以上（業界平均15-20%）。編集テンポ・尺配分・ループ繋ぎ設計の総合評価
- **KPI-05: Loop Rate（自動ループ再生回数）**：目標平均1.6倍以上（5/25の新KPI）。冒頭3秒＝最後3秒の繋ぎ設計・2周目発見要素の実装度
- **KPI-06: 納品後修正依頼件数**：目標月0件。書き出し前4軸チェック・下書きテストアップロード・実機プレビュー録画の3段ゲートの効果。修正の実務コスト（1件平均2時間）と直結
- **KPI-07: 媒体側再圧縮後の可読性維持率**：目標100%（テロップ・色・鮮明さ）。下書きテストアップロードによる媒体圧縮劣化の事前検知率

### 6. 出荷前セルフチェックリスト（5項目）

書き出し前・納品前に必ず全項目を通過すること。1項目でも未達なら書き出し・納品NGとする最終ゲート。

- [ ] **チェック①：Visual Hook 0.5倍速検証** — 0.0-1.5秒を0.5倍速で1コマ再生し、6大視覚要素（動き/色/顔/数字/意外性/パターン破壊）のうち3つ以上が0.0-0.5秒に発火しているか目視確認。3つ未満なら作り直し
- [ ] **チェック②：音声二段管理（dBFS+LUFS）** — ピーク-12dB±3dB（ナレ）／-24dB以下（BGM）／-18dB以下（SE）の3層差化AND統合ラウドネス-14〜-16LUFS。ラウドネスメーターの数値スクリーンショットを納品時に添付
- [ ] **チェック③：テロップ3条件AND充足＋専門用語照合** — 60pt以上・コントラスト4.5:1以上・1.5秒以上表示のAND充足に加え、建設用語カスタム辞書（足場/型枠/鉄筋/躯体/玉掛け）で一括置換チェック済み。現場社員1名の目視確認済み
- [ ] **チェック④：TikTok非公開下書きテストアップロード** — 媒体側再エンコード後の実機状態を確認し、①テロップ縁の滲みなし②暗部ブロックノイズなし③1440×2560鮮明さ維持④UIオーバーレイ後の可読性維持の4点を実機で確認
- [ ] **チェック⑤：カラーグレーディング基準カット照合＋アンビエント音-30dB残し** — グレーディング前後の基準カットを並べて①安全ベスト蛍光色維持②肌色血色維持③空・コンクリ自然色の3点目視照合。屋外案件は現場アンビエント音を-30dBで薄く残しているか確認
