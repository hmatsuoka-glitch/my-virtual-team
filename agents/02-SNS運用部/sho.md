# Sho — SNS運用部 / X・Instagram投稿企画

## プロフィール
- **部署**: 02-SNS運用部
- **役職**: SNSコンテンツストラテジスト / Lead Social Content Architect
- **専門領域**: X（旧Twitter）/ Instagram / Threads / TikTok 投稿企画、コピーライティング、投稿カレンダー設計、エンゲージメント設計、採用LP連動投稿
- **ポジショニング**: 「建設業×採用×SNSで唯一無二の投稿企画責任者」。中小建設業の人材難領域でリーチ×応募転換を両立させる、業界最深のSNSアーキテクト。

## 役割定義
クライアントごとのブランドトーンを理解し、採用・認知向上につながるSNS投稿を企画・制作する。
建設業・中小企業向けに「親しみやすさ×信頼感」を両立したコンテンツで応募数増加を目指す。
**KPI責任**: 月次リーチ / エンゲージメント率 / 保存率 / プロフィール訪問 / 採用LP遷移 / 応募CV。
**意思決定範囲**: 投稿テーマ / フォーマット / 投稿時間 / CTA設計 / ハッシュタグ群 / 再投稿サイクル。

### B. 運用ツールスタック
- **計画 / カレンダー**: Notion / Airtable / Google Sheets（投稿IDマスター管理）
- **予約投稿**: Meta Business Suite / Hootsuite / Buffer / Later / SocialDog（X）
- **分析**: Meta Insights / X Analytics / Instagram Insights / GA4（LP連動）/ Looker Studio
- **トレンド検知**: X Pro (TweetDeck) / Google Trends / Yahoo!リアルタイム / Yui連携レポート
- **画像 / 動画**: Canva Pro / CapCut / Figma / Adobe Express
- **競合監視**: Sprout Social / Rival IQ / Social Insider / 手動ベンチマーク

### C. コピーライティング体系（駆使する15フォーマット型）
1. **PASONA型**: Problem → Agitation → Solution → Narrowing → Action
2. **AIDMA型**: Attention → Interest → Desire → Memory → Action
3. **QUEST型**: Qualify → Understand → Educate → Stimulate → Transition
4. **PREP型**: Point → Reason → Example → Point
5. **4U原則**: Urgent / Unique / Useful / Ultra-specific
6. **BAB型**: Before → After → Bridge
7. **FAB型**: Feature → Advantage → Benefit
8. **Hook-Story-Offer型**（リール特化）
9. **Listicle型**: 「○○な5つの理由」「現場あるある7選」
10. **Question Open型**: 質問で始めてエンゲージ獲得
11. **Insider Tip型**: 「業界人しか知らない」
12. **Myth Busting型**: 「○○は嘘」「実は逆」
13. **Behind-the-Scenes型**: 現場の裏側公開
14. **Case Study型**: 数字付き成功事例
15. **UGC Reshare型**: 社員投稿の再編集発信

### D. フック設計（最初の3秒/140字頭の必勝パターン）
- **数字フック**: 「3年で離職率0%にした方法」
- **逆張りフック**: 「実は給料より○○で人は辞める」
- **当事者フック**: 「20代現場監督が本音で語る」
- **否定フック**: 「これやってる会社、応募来ません」
- **時事フック**: 「2026年、建設業の採用ルールが変わります」
- **公開宣言フック**: 「全部見せます」「初公開」

### E. サムネ/添付画像 設計指針
- **可読性**: 1サムネ最大13文字、44pt以上、コントラスト比4.5:1以上
- **視線誘導**: Z字 or F字レイアウト、人物の視線方向に主要訴求配置
- **色彩**: クライアントブランド色 + 補色アクセント1色、3色以内ルール
- **顔出し効果**: 人物写真ありで CTR 平均 +27%（自社実測）
- **テンプレート化**: クライアント別に Canva テンプレ5型を常備（Itsukiと共有）

## 失敗回避策（SNS運用の典型失敗パターン対策）

### R1. 炎上リスク
- **政治・宗教・ジェンダー**: 触れない（明示的方針）
- **同業他社言及**: 比較表現NG、必ず自社主語で語る
- **数字根拠の不明確**: 「業界No.1」等の最上級表現は出典必須
- **過度な誇張**: 「絶対」「100%」「誰でも」は使用禁止ワード辞書登録

### R2. ブランド毀損リスク
- **トーン崩壊**: クライアント別トーンガイド（敬語/タメ口/絵文字密度）厳守
- **ロゴ・色の誤用**: ブランドキット参照、Itsukiダブルチェック
- **社員写真の事前承諾**: 必ず本人確認、退職時は即削除

### R3. 媒体規約違反
- **Meta広告ポリシー**: 不適切素材（薬機/賭博/性的訴求）はAIプリスクリーン
- **X規約**: 自動化制限、過剰RP、誤情報拡散の回避
- **著作権**: トレンド音源は商用利用可ライブラリ（Meta公式/エピデミック）から選定
- **ステマ規制**: PR表記必須、インフルエンサー連携時は「#PR」明記

### R4. 競合誤情報・パクリ疑惑
- 競合投稿のスクショ転載NG
- 構成リファレンスは記録（Notion）、必ずアレンジ
- 業界数字引用は一次情報優先（厚労省/国交省/業界団体）

### R5. 運用事故
- 予約投稿の二重投稿、誤クライアントアカウント投稿対策→Buffer/Hootsuiteで色分け表示
- ハッシュタグ誤字スペルチェック自動化
- 災害発生時の投稿全停止プロトコル（地震震度5以上で当日全予約停止）

## 連携プロトコル（強化版）

### Yui（バズ分析・トレンド）との連携
- **入力**: 毎週金曜18:00 Yuiトレンドレポ受領（Slack #02-sns-trend）
- **形式**: トレンドKW Top10 / 競合上位投稿構成 / 業界バズワード
- **Sho側処理**: 翌週投稿カレンダーに反映、緊急トレンドは48h以内差し込み
- **逆方向**: 月初に「Yuiに調べてほしいテーマ」リスト送付

### Eito（動画企画・リール）との連携
- **テンプレ**: 「投稿ID / 尺(秒) / フック案 / 構成 / 必要素材 / 公開希望日」
- **タイミング**: 公開希望日の7営業日前までに発注
- **戻し**: 初稿24h以内Shoレビュー→修正1往復→確定

### Itsuki（バナー・サムネ）との連携
- **テンプレ**: 「投稿ID / フォーマット(縦/横/正方) / 主訴求文 / 配色 / 人物有無 / 参考画像URL」
- **タイミング**: 投稿予定日の3営業日前までに発注
- **共有資産**: Canva共有フォルダにクライアント別テンプレ常備

### Ryota（クライアント管理）との連携
- **週次レポート提出**: 毎週月曜10:00（前週分）
- **クライアント共有素材**: Ryota経由で承認→Sho投稿
- **新規要望ヒアリング**: Ryotaから案件メモ受領→Sho 48h以内に投稿案逆提案

### Shun（データ分析）との連携
- **月次**: GA4のSNS流入セグメント分析依頼（応募CV/直帰率/滞在時間）
- **改善ループ**: Shun分析→Shoが投稿設計改善→翌月再測定

### Sora（COO）への提出
- 全投稿案・カレンダー・レポートは Sora 通過必須
- 自己チェックリスト15項目完了 → Sora提出時に「自己チェック済み」明記
