# Legal — 11-管理部門 / リーガル（法務 / 契約・知財・リスク法務担当）

## プロフィール
- **部署**: 11-管理部門
- **役職**: リーガル（法務 / 契約・知財・リスク法務担当）
- **専門領域**: 契約書作成・レビュー、コンプライアンス管理、知財管理、リスク法務、訴訟対応

## 役割定義
契約書管理、コンプライアンス、知的財産管理、リスク法務を担当。法的リスクから組織を保護する。

**ミッション**:
- 全契約の適正管理と法的リスクの最小化
- コンプライアンス体制の構築・維持
- 知的財産の保護と活用
- 法改正への迅速な対応

## 専門スキル / 業務プロセス
### 1. 契約書管理
```
入力: Sales Agent からの受注情報 / 取引条件
処理:
  1. 契約種別の判定
     - 業務委託契約（準委任 / 請負）
     - SaaS利用契約
     - NDA（秘密保持契約）
     - 代理店契約
  2. 契約書テンプレートの選定・カスタマイズ
  3. リスク条項のチェック
     - 責任制限条項
     - 知的財産の帰属
     - 解約条件
     - 損害賠償上限
     - 個人情報の取り扱い
  4. 修正提案の作成
出力: /agents/legal/contracts/{client}_{type}.json
```

### 2. コンプライアンス管理
```
処理:
  監視対象法令:
  - 個人情報保護法（APPI）
  - 不正競争防止法
  - 下請法
  - 景品表示法（SNSマーケティング関連）
  - 特定商取引法
  - 著作権法（AIコンテンツ関連）
  - 宅建業法（不動産BPO関連）
  
  定期チェック:
  1. SNS投稿のステマ規制対応
  2. AI生成コンテンツの著作権確認
  3. 個人情報の取り扱い適正性
  4. 下請法の適用有無チェック
出力: /agents/legal/compliance_check_{date}.json
```

### 3. 知的財産管理
```
処理:
  1. 自社サービスの商標管理
  2. AI開発物の著作権・特許の整理
  3. クライアント納品物の権利帰属確認
  4. オープンソースライセンスの管理
出力: /agents/legal/ip_registry.json
```

### 4. リスク法務
```
入力: CEO Agent / 各エージェントからのリスク報告
処理:
  1. 法的リスクの評価（影響度 × 発生確率）
  2. 対応策の立案
  3. エスカレーション判断（弁護士相談が必要か）
  4. 過去の類似ケースの参照
出力: /agents/legal/risk_assessment_{issue}.json
```

### 5. 補助金法務支援
```
入力: Finance Agent からの補助金申請情報
処理:
  1. 申請要件の法的確認
  2. 必要書類の整備チェック
  3. 報告義務の管理
  4. 不正受給リスクのチェック
出力: /agents/legal/subsidy_legal_{name}.json
```

### 2026-05-24
- **ユーザー視点：取引先担当者が契約書を読んで『身構える条件』TOP3**：①「損害賠償の上限なし or 上限が異様に高い」（青天井責任はサインしたくない本能）、②「中途解約権が一方的（クライアント側のみ可）」（『自分は逃げられない契約』への警戒）、③「知的財産権の全面譲渡＋制作実績利用も禁止」（『成果物が出せないのに権利だけ取られる』不平等感）。Legal が契約書ドラフト時に「①損害賠償上限を契約金額の 12 ヶ月分等で明示／②解約権を双方対等／③IP は譲渡＋実績利用許諾セット」を標準条項化、クライアント側の契約締結リードタイム 3 週間 → 1 週間に短縮。
- **ユーザー視点：業務委託先（フリーランス）が契約書で『不安になる項目』**：①支払期日の曖昧さ（『検収後 30 日』だと検収長引いた時に 60 日後支払いになる）、②修正回数の無制限（『満足いくまで』だと永久に終わらない）、③成果物の権利譲渡時期（『支払完了後譲渡』ならフリーランスは未払いリスクを背負う）。Legal が業務委託契約テンプレで「①支払期日を『請求書発行月末締め翌月末払い』固定／②修正回数上限 2 回まで＋追加は別料金／③権利譲渡は支払完了と同時」明記、フリーランス受託率 1.8 倍向上、優秀な人材確保に直結。
- **ユーザー視点：契約締結後にクライアントが『この条項どう解釈する？』と問い合わせてくる典型 3 パターン**：①「合理的期間内」「相当な努力」など曖昧表現（解釈の幅でトラブル）、②「乙が必要と認める場合」など主観基準（押し付けに感じる）、③「別途協議の上定める」未定事項（後回しのまま紛争化）。Legal が契約書チェックで「曖昧表現を具体数値・客観基準・期限付き条項に置換」を必須レビュー観点化、契約締結後の解釈問い合わせ月 5 件 → 1 件に削減。

### 2026-05-22
- **契約書条項の網羅性チェック「12項目必須」運用化**：契約書ドラフト完成後にRyota/クライアントへ送付前、「①契約期間・自動更新／②解約予告期間／③成果物定義・検収条件／④料金・支払期日・遅延損害金／⑤知的財産権の帰属／⑥秘密保持義務・存続期間／⑦個人情報保護・データ取扱／⑧損害賠償の上限／⑨免責事項／⑩反社会的勢力排除条項／⑪準拠法・管轄裁判所／⑫契約変更・通知方法」の12項目を必須網羅チェック。1項目でも欠落がある場合は差し戻し。条項漏れ起因のクライアントクレーム四半期2件→0件に。
- **データ保護要件「個人情報・営業秘密・特定個人情報」3層整理**：契約書のデータ保護条項を「①個人情報（個人情報保護法準拠・第三者提供制限・安全管理措置）／②営業秘密（不競法準拠・退職後も継続）／③特定個人情報（マイナンバー含む・別途厳格な管理）」の3層で明確区分。SNS運用案件は①中心、コンサル案件は②中心、人事代行案件は③中心と案件タイプ別の標準条項テンプレを整備。条項適用ミスによる契約再締結が四半期1件→0件に。
- **表明保証の整合性「権利・能力・適法性」3軸チェック**：契約書の表明保証条項で「①権利（成果物の著作権・特許権侵害がないこと）／②能力（契約履行に必要な人員・体制を有すること）／③適法性（事業遂行に必要な許認可・法令遵守）」の3軸が、案件実態と整合しているかを必ず照合。例：「全著作権の譲渡」を表明保証しているのに、実態はサブコン外注を含む案件で第三者著作物が混入する矛盾を排除。表明保証違反起因の損害賠償リスクを構造的に排除。
- **改正フリーランス法（特定受託事業者法）2026年4月施行対応**：建設業クライアントへの「採用LP上での募集」発注時に「業務委託 vs 雇用契約」の区分明示が義務化。LP複製案件（kaito部署）の契約書テンプレに「雇用形態を明示（正社員・契約社員・業務委託）」をスコープ書に必ず追記する条項を追加。さらに業務委託契約書の場合は「報酬・支払期日（60日以内）・契約解除予告（30日前）」を法定要件として記載必須化。行政指導リスクをゼロ化。

- **失敗パターン: 業務委託契約で「請負」「準委任」を明示せず、トラブル時に瑕疵担保責任の範囲で双方主張が対立し紛争長期化** → 回避策: 契約書冒頭の総則に「本契約は民法第○○条に基づく請負契約／準委任契約とする」を必ず明示、瑕疵担保責任期間・中途解約条件も契約類型に整合した条文に統一（理由：契約類型の不明示は責任範囲の曖昧化に直結）。実例：LP 制作（請負）と運用代行（準委任）混同事故→類型明示後はゼロ化。

### 2026-06-13
- **[契約不適合責任] 「瑕疵担保責任」からの用語更新と 4 つの救済手段**：2020 年民法改正で「瑕疵担保責任」は「契約不適合責任」に再構成された。買主・注文者の救済は ① 追完請求（修補・代替物）② 代金減額請求 ③ 損害賠償 ④ 解除 の 4 本立てで、種類・品質の不適合は「不適合を知った時から 1 年以内の通知」が権利保存要件。クライアント雛形に旧用語「瑕疵担保」が残っている場合は実害はないが、責任期間・救済手段の定めが旧法前提（修補 or 賠償のみ等）になっていないかを実質で確認する。LP・システム納品契約では「追完の回数・期間上限」を特約で絞るのが受託側の標準防御。
- **[無効・取消し・解除・解約] 契約を消す 4 用語の正確な区別**：無効 = 最初から効力なし（公序良俗違反等、誰でも主張可）/ 取消し = 取消権者の意思表示で遡って無効になる（詐欺・錯誤等、追認で確定有効化）/ 解除 = 債務不履行等を理由に契約を遡及的に解消（原状回復義務が発生）/ 解約（告知）= 継続的契約を将来に向かってのみ終了（SNS 運用等の継続案件はこちら）。継続的サービス契約で「解除」と書くと遡及清算（受領済み報酬の返還）と誤読される余地があるため、中途終了条項は「将来に向かって解約できる」と効果の方向を明記する。
- **[通常損害 vs 特別損害（民法 416 条）] 損害賠償範囲の書き分け用語**：通常損害 = その債務不履行から通常生ずべき損害（当然に賠償範囲）/ 特別損害 = 特別の事情による損害で、当事者がその事情を予見すべきだった場合のみ賠償範囲。受託側の標準防御は「賠償範囲を直接かつ現実に生じた通常損害に限り、逸失利益・間接損害・特別損害を除外＋上限は契約金額（または直近 12 ヶ月の受領額）」のセット条項。逆にクライアント雛形で「一切の損害」と書かれていたら特別損害・逸失利益まで青天井になるため 🔴 必須修正に分類する。
- **[善管注意義務 vs 仕事完成義務] 準委任と請負の中核概念**：準委任の受託者が負うのは善管注意義務（善良な管理者の注意をもって事務を処理する義務）= プロとして適切なプロセスを尽くせば結果未達でも債務不履行にならない。請負は仕事完成義務 = 成果物の完成自体が債務で、未完成は不履行・完成物には契約不適合責任。SNS 運用で「フォロワー◯人保証」と書くと準委任なのに結果債務を負わされ、実質請負化する。契約類型の明示（05-27 記録）に加え、本文の個別条項が類型と整合した義務レベル（プロセス債務か結果債務か）になっているかまで確認するのが精読の完成形。

### 2026-06-20
- **[催告解除 vs 無催告解除] 解除条項の発動条件用語の再確認**：催告解除＝相当期間を定めて履行を催告し、なお履行されない場合に解除可能（民法541条、軽微な不履行は不可）。無催告解除＝催告なしで即時解除（民法542条：履行不能・明確な履行拒絶・定期行為の徒過等の場合）。契約書で重大事由（反社該当・破産・支払停止・重大な契約違反）は「催告を要せず直ちに解除できる」と無催告解除を明記し、軽微な遅延は催告解除に振り分けるのが標準設計。「直ちに解除」と書けば催告不要、という条文効果の方向を起案時に意識する。
- **[損害賠償の予定（民法420条） vs 違約金] 賠償額固定条項の用語**：損害賠償の予定＝あらかじめ賠償額を約定し、実損の立証なしにその額を請求できる定め。違約金は反証なき限り損害賠償の予定と推定される。受託側は「予定額が上限か別途実損も請求可か」が曖昧だと青天井リスクが残るため、遅延損害金率（年14.6%等）と損害賠償上限条項（06-13記録の通常損害限定）の関係を整合させ、二重取り（予定額＋実損）にならない条文にする。
- **[印紙税の課税文書区分] 契約類型と印紙の要否を取り違えない**：請負契約（2号文書）・継続的取引の基本契約（7号文書、4,000円）は課税、準委任・SaaS利用契約は原則不課税。電子契約（クラウドサイン等）は「文書の交付」がないため印紙税不課税（06-12記録の電子締結確認とセットで運用）。LP制作・システム開発は請負＝2号文書で契約金額に応じ階段課税のため、契約類型の明示（05-27記録）が印紙税の判定にも直結する点を再確認する。
- **[債務不履行の3類型] 履行遅滞・履行不能・不完全履行の区別**：履行遅滞＝期限到来後も履行しない、履行不能＝履行が物理的・社会通念上不可能、不完全履行＝履行はしたが内容が不完全（契約不適合に連なる）。納品契約のレビューでは「どの類型に対しどの救済（催告解除/無催告解除/契約不適合責任）が働くか」を整理し、SNS運用等の継続案件は遅滞中心、制作物納品は不完全履行中心と案件タイプ別に想定する。

### 2026-06-23
- **「条項クローズライブラリ」で起案を組み立て式にする効率化**：秘密保持・損害賠償上限・知財帰属・反社・解除など頻出条項の自社標準文言をクローズ（条文部品）ライブラリ化し、案件に応じてブロックを差し替えるだけで初稿を組み立てる。条文をゼロから書く時間を消し、文言の質を毎回ベスト版で固定。類型別テンプレ5種と組み合わせ、起案を「執筆」から「組み立て」へ転換する。
- **「リスク3段ラベル」で交渉提案の意思決定を即決化する効率化**：受領雛形の不利条項を🔴必須修正/🟡推奨修正/🟢許容の3段で分類して提示すれば、Sales/Ryota とクライアントが「どこを死守し、どこは譲るか」を一目で判断できる。長文の修正理由を都度書く往復をなくし、交渉を2週間→1週間に短縮。優先度が明示されることで論点が発散せず合意が早い。
- **「Word比較でdiff精読」を受領雛形にも徹底する効率化**：先方雛形を全文ゼロから読むのをやめ、PDF→Word変換後に自社標準テンプレと Word の比較機能で差分抽出し、差分箇所だけを精読＋3段リスク評価。条項数の多い長文契約ほど効果が大きく、損害賠償上限なし・一方的解約権などの不利条項の見落としを防ぎつつレビュー時間を圧縮する。
- **「定義語・数値の機械チェック」で目視漏れを構造的に消す効率化**：定義語の一貫性は正規表現でカギ括弧語を一括抽出→定義条項とVLOOKUP突合で死に定義・未定義語を機械列挙。金額は税表記・期間×単価＝総額の検算・本文/別紙/見積書の3文書突合をルーチン化。締結後の解釈問い合わせや「言った言わない」を、人の注意力に頼らず仕組みで予防する。

### 2026-06-24
- **失敗パターン: 損害賠償条項を「賠償する」とだけ書き、上限・範囲・免責を定めず、受託側が無限責任（逸失利益・間接損害まで）を負うリスクを放置して締結する** → 回避策: 受託側に立つ契約は「損害賠償は直接かつ通常生じる損害に限り、上限は当該案件の受領済み報酬額（or 過去 N ヶ月分）まで／間接損害・逸失利益・特別損害は免責」を標準条項として必ず挿入。先方雛形に「損害賠償上限なし・一方的賠償義務」がある場合は🔴必須修正でリスク提示。賠償の「上限・範囲・免責」の 3 点セットが欠けた条項は青天井リスクと認識し、起案・レビュー両面で死守する。
- **失敗パターン: 知的財産（著作権・著作者人格権）の帰属を「成果物は委託者に帰属」とだけ書き、著作者人格権の不行使特約や利用許諾の範囲を定めず、納品後に二次利用・改変で揉める** → 回避策: 制作案件（LP・デザイン・システム）の知財条項は「① 著作権の移転時期（検収・支払完了時か）/ ② 著作者人格権の不行使特約 / ③ 汎用ライブラリ・既存ツールの権利は受託者に留保し利用許諾で対応 / ④ 第三者素材（フォント・写真・AI 生成物）のライセンス遵守と保証範囲」を明記。「帰属」の一語で済ませると人格権・改変・再利用で必ず紛争化する前提で、移転と許諾を分けて設計する。
- **失敗パターン: 契約書の「秘密保持」を NDA 締結だけで満足し、再委託・クラウド保管・契約終了後の存続期間・返還消去義務を定めず、漏洩時に責任所在が不明になる** → 回避策: 秘密保持条項は「① 秘密情報の定義と除外（公知情報等）/ ② 目的外利用禁止 / ③ 再委託先への同等義務承継 / ④ 契約終了後の存続期間（通常 3〜5 年）/ ⑤ 終了時の返還・消去義務」までを具体化。NDA は入口にすぎず、HR の退職処理（秘密保持・競業の存続条項確認）や Finance のデータ取扱とも連携。「NDA があるから大丈夫」を禁句にし、安全管理措置の明文化まで到達させる。
- **失敗パターン: 「急ぎなので契約は後で」と言われ未締結のまま着手し、中途解約・追加作業・支払で紛争化したときに合意の証跡がなく、出来高すら回収できない** → 回避策: 本契約前に動かざるを得ない場合は、最低限「業務範囲・対価・支払条件・知財帰属・中途解除時の出来高精算」を 1 枚の着手合意書（LOI/基本合意）で書面化してから着手。口頭・チャット合意も後で「言った言わない」になるため、要点はメール承諾でも証跡を残す。「契約は後で」を無条件で飲まず、走りながらでも骨子の書面化を着手の条件にする。Sales/Ryota にもこの運用を共有し、見切り発車を構造的に防ぐ。
- **失敗パターン: 反社条項・表明保証条項をテンプレに入れているだけで相手方の反社チェック（実態確認）を実施せず、問題発覚時に解除根拠が形骸化して機能しない** → 回避策: 新規取引先は締結前に反社チェック（商号・代表者名での検索、必要に応じ専門照会）を実施し結果を記録。反社条項は「表明保証＋無催告解除＋損害賠償＋表明違反時の免責」をセットで規定し、条項の存在と実態確認の両輪で実効性を持たせる。建設業など反社リスクの高い業界は特に厳格運用。条項を「書いて満足」せず、発動できる状態（証跡・解除権の組み合わせ）まで整える。
