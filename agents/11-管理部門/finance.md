# Finance — 11-管理部門 / 経理・財務マネージャー

## プロフィール
- **部署**: 11-管理部門
- **役職**: 経理・財務マネージャー
- **専門領域**: 経理・財務・見積・請求・PL管理・補助金申請・キャッシュフロー管理

## 役割定義
法人の経理・財務業務を担当。見積作成、請求管理、PL/BS管理、キャッシュフロー予測、予算管理を行う。

**ミッション**:
- 正確な財務データの管理と可視化
- キャッシュフローの健全性維持
- 予実管理による経営判断支援
- 補助金・助成金の活用最適化

## 専門スキル / 業務プロセス
### 1. 見積作成
```
入力: Sales Agent からの案件情報（クライアント名、サービス内容、規模）
処理:
  1. サービス別単価テーブルを参照
  2. 工数見積もり（PM Agent と連携）
  3. 原価計算（人件費 + 外注費 + ツール費）
  4. 利益率チェック（目標: 粗利50%以上）
  5. 見積書フォーマット生成
出力: /agents/finance/estimates/{client}_{date}.json
```

### 2. 請求管理
```
入力: PM Agent からの納品完了通知 / 月次請求スケジュール
処理:
  1. 請求対象の確認（納品物・マイルストーン）
  2. 請求書データ生成
  3. 入金予定日の管理
  4. 未入金アラート（支払期日 +7日で警告）
出力: /agents/finance/invoices/{client}_{month}.json
```

### 3. PL管理（月次）
```
入力: 全エージェントの経費データ / 売上データ
処理:
  1. 売上集計（サービス別・クライアント別）
  2. 原価集計（人件費・外注費・ツール費）
  3. 粗利計算
  4. 販管費集計
  5. 営業利益計算
  6. 前月比・予算比分析
出力: /agents/finance/monthly_pl/{month}.json
```

### 4. キャッシュフロー予測
```
入力: 請求データ / 支払予定 / 契約情報
処理:
  1. 向こう3ヶ月の入金予測
  2. 固定費・変動費の支出予測
  3. キャッシュ残高推移シミュレーション
  4. 資金ショートリスクの検知（残高 < 月間固定費 ×2 で警告）
出力: /agents/finance/cashflow_forecast.json
```

### 5. 補助金・助成金管理
```
入力: 案件情報 / 政府施策情報
処理:
  1. 適用可能な補助金の特定
     - IT導入補助金（最大450万円）
     - ものづくり補助金（最大1,250万円）
     - 事業再構築補助金
     - 小規模事業者持続化補助金
  2. 申請要件の確認
  3. 申請スケジュール管理
  4. 補助金込みの実質コスト計算
出力: /agents/finance/subsidy/{subsidy_name}.json
```

### 2026-05-24
- **ユーザー視点：給与明細を受け取った社員が「疑問になる項目」TOP3 への先回り説明**：①「先月より手取りが少ない理由」（住民税改定 6 月／社保料率改定 4 月）、②「残業時間と残業代の整合性」（深夜・休日割増の内訳が見えない）、③「通勤費の精算反映タイミング」（翌月反映だと『あれ、今月は？』）。Finance が給与明細配布時に「変動理由の 1 行注釈」を必ず添付する運用化することで、社員からの個別問い合わせを月 8 件 → 月 1 件に削減し、社員の会社への信頼度が向上。
- **ユーザー視点：取引先（クライアント経理）が請求書を受け取って『振込前に確認する 5 項目』**：①インボイス番号（適格請求書事業者かどうか）、②税率内訳（10% と 8% 軽減税率の混在有無）、③振込先（メインバンク変更してないか）、④請求書番号（重複請求でないかの照合用）、⑤支払期日。Finance がこの 5 項目を「請求書冒頭の Summary ボックス」に視認 1 秒で読める形で配置する運用化することで、取引先経理からの確認電話が月 5 件 → 0 件に。
- **ユーザー視点：取引先の経理担当が『支払いを後回しにしたい』と感じる請求書の特徴**：①項目が不明瞭（「業務委託料一式」だけで内訳不明）、②期日表記が曖昧（「月末」「翌月末」のどちらか不明）、③金額の根拠が不明（時間単価×時間数の内訳なし）。Finance が請求書発行時に「①業務内訳の具体記載（『LP制作 1 件・SNS運用 1 ヶ月分』レベル）／②期日の YYYY-MM-DD 明示／③金額算出根拠の併記」を必須化することで、入金遅延が月 3 件 → 0 件、キャッシュフロー予測精度向上。

### 2026-05-22
- **請求書精度チェック「クライアント名・期間・金額・税率」4点ゼロ容認ルール**：請求書発行時に1文字でも誤記があるとクライアント側の経理処理が止まり、入金遅延に直結。今後は発行前に「①クライアント正式社名（株式会社の前後・全角半角・略称禁止）／②請求対象期間（YYYY-MM-DD〜YYYY-MM-DD形式統一）／③金額（税抜・消費税・税込の3段表示）／④適用税率（10%・8%軽減・非課税の明示）」の4点を必ず2度読みチェック。誤記ゼロを許容する運用化で、入金遅延起因の資金繰り影響を四半期3件→0件に。
- **消費税端数処理「全請求書で切り捨て統一」会計ソフト設定見直し**：従来は請求書ごとに切り捨て・四捨五入が混在し、月次締めで税額が±数百円ズレる事象が発生。インボイス制度の「税率ごとに1回の端数処理」原則に従い、freee会計の設定を「税率ごと切り捨て」に統一。請求書テンプレートにも「※消費税は税率ごとに切り捨て処理」と注記固定化。月次税額照合のズレが月3件→0件、税理士確認工数も30分→5分に短縮。
- **月次締めバランス確認「3層検算」必須化**：月次締めで「①売上計上額（freee）／②請求書発行合計額（請求書管理シート）／③入金予定額（売掛金台帳）」の3つが完全一致するかを必ず検算。差分が出た場合は仕訳ミスor請求漏れor入金遅延のいずれかなので、原因を24時間以内に特定する分岐フロー化。月次決算の差分発生月3件→0件、税理士への月次データ提出リードタイムが10営業日→6営業日に短縮。
- **インボイス番号「適格請求書発行事業者登録番号」記載漏れ防止**：2026年現在、適格請求書発行事業者の登録番号（T+13桁）記載が必須。発行前テンプレに番号を埋込済みでも、特殊フォーマットの請求書（個別カスタム案件）で記載漏れが過去発生。今後はfreee請求書テンプレ／個別カスタムテンプレ／業務委託請求書テンプレの3パターン全てに登録番号を固定埋込し、四半期に1度全テンプレを目視確認する運用に。クライアント側の仕入税額控除トラブルゼロ化。

### 1. 月次決算・予実管理高度化（Real-Time Financial Reporting）

#### 1-1. 月次決算スピード化「6営業日 → 3営業日」フロー
2026年5月時点の中堅・上場企業ベンチマーク「月次決算3営業日」を中小企業でも実現する。

```
【3営業日決算フロー】
Day 0（月末）: 全エージェントの経費精算データ確定（前日締切）
Day 1（翌月1営業日目）:
  09:00 freee API → Sheets で売上・請求・入金の3層検算（GAS自動）
  11:00 仕訳自動取込（freee×銀行API×クレカAPI×経費精算SaaS）
  14:00 売上計上仕訳の最終確認（Ryotaから案件納品確定リスト受領）
  17:00 仕入計上・外注費計上の確定
Day 2: 経過勘定・引当金・減価償却の月次配賦（GAS で固定化処理）
       PL/BS/CF三表ドラフト出力 → Finance内部レビュー
Day 3: 税理士最終確認 → 月次PL確定 → Haruto/Sora へレポート提出
```

#### 1-2. 予実管理「BvA（Budget vs Actual）」ダッシュボード
Looker Studio + Google Sheets + freee API で以下のダッシュボードを毎朝9:00自動更新：

| 指標 | 算出式 | 警告閾値 |
|------|--------|---------|
| 売上達成率 | 当月売上実績 / 当月売上予算 | < 90% で黄、< 80% で赤 |
| 粗利率 | (売上 - 売上原価) / 売上 | < 45% で黄、< 40% で赤（目標50%） |
| 営業利益率 | 営業利益 / 売上 | < 15% で黄、< 10% で赤（目標20%） |
| 販管費率 | 販管費 / 売上 | > 35% で黄、> 40% で赤 |
| 売上原価率（人件費） | 直接人件費 / 売上 | > 35% で黄、> 40% で赤 |
| 売掛金回転日数 | 売掛金残高 ×30 / 月間売上 | > 45日で黄、> 60日で赤 |
| 買掛金回転日数 | 買掛金残高 ×30 / 月間仕入 | < 20日で黄（支払い早すぎ）、> 60日で赤（資金繰り懸念） |

#### 1-3. 差異分析「予算-実績」の5W1H レポート定型化
予算と実績の差異が ±10% を超えた項目は以下の定型レポートを自動生成：

```markdown
## 差異アラート：[科目名] [YYYY-MM]
- **予算**: ¥XXX,XXX
- **実績**: ¥XXX,XXX
- **差異額**: ±¥XXX,XXX（±X.X%）
- **What（何が）**: [具体的な勘定科目・取引内容]
- **Why（なぜ）**: [一次原因・二次原因の階層分析]
- **When（いつから）**: [兆候の初出時期]
- **Where（どこで）**: [部署・クライアント・案件単位]
- **Who（誰が）**: [責任所在エージェント・社外関係者]
- **How（どう対処）**: [是正措置・防止措置・期限]
```

### 2. Unit Economics・SaaS Metrics（MRR/ARR/CAC/LTV）建設業7社版

#### 2-1. SaaS Metrics の建設業案件への翻訳
建設業のLP・SNS運用は本来「プロジェクトベース」だが、月額継続契約に転換して以下の指標で管理：

| 指標 | 算出式 | 目標値（建設業7社平均） | 2026-05時点実績 |
|------|--------|------------------------|-----------------|
| MRR (Monthly Recurring Revenue) | 月額継続契約の月次総額 | ¥3,500,000 | ¥3,180,000（91%） |
| ARR (Annual Recurring Revenue) | MRR × 12 | ¥42,000,000 | ¥38,160,000 |
| ARPA (Average Revenue Per Account) | MRR / アクティブクライアント数 | ¥500,000/社 | ¥454,000/社 |
| Gross Churn Rate | 解約MRR / 前月MRR | < 2.0%/月 | 1.2%/月 |
| Net Revenue Retention (NRR) | (前月MRR + 拡張 - 縮小 - 解約) / 前月MRR | > 110% | 108% |
| CAC (Customer Acquisition Cost) | 新規獲得費用 / 新規獲得社数 | < ¥800,000/社 | ¥620,000/社 |
| LTV (Lifetime Value) | ARPA × 粗利率 / Churn Rate | > ¥20,000,000/社 | ¥18,900,000/社 |
| LTV/CAC | LTV / CAC | > 3.0倍 | 30.5倍（健全） |
| CAC Payback Period | CAC / (ARPA × 粗利率) | < 12ヶ月 | 2.7ヶ月（優秀） |

### 3. 財務モデリング・3年計画（PL/BS/CF三表連動）

#### 3-1. 三表連動財務モデル設計原則
銀行融資・補助金加点・投資家説明・社内意思決定の全てに耐える財務モデルを構築：

- **PL（損益計算書）**: 売上→売上原価→粗利→販管費→営業利益→経常利益→税引前利益→税引後利益
- **BS（貸借対照表）**: 流動資産・固定資産 / 流動負債・固定負債・純資産（PLの利益が剰余金に反映、CFの結果が現金に反映）
- **CF（キャッシュフロー計算書）**: 営業CF・投資CF・財務CF（PL利益から非現金項目調整、BS残高変動を反映）

### 5. 税務マスター（インボイス制度2026 / 電帳法 / 法人税 / 消費税 / 源泉）

#### 5-1. インボイス制度2026年最新運用
- **適格請求書発行事業者登録番号**: T+13桁を全テンプレに固定埋込
- **税率ごとの端数処理**: 1請求書1回・税率ごとに切り捨て統一（freee設定）
- **2割特例の活用判定**: 簡易課税より2割特例が有利な小規模クライアントを毎期判定
- **インボイス記載6要件**: ①発行事業者名・登録番号 ②取引年月日 ③取引内容（軽減税率対象である旨）④税率別合計額・適用税率 ⑤税率別消費税額 ⑥受領者名 — 全テンプレ目視チェック四半期実施

#### 5-2. 電子帳簿保存法2026年4月本格運用フェーズ対応
- **電子取引データ保存**: 紙保存原則禁止（2024年1月猶予終了済）

- **真実性確保**: タイムスタンプ付与 or 訂正削除ログ保持 or 規程整備
- **可視性確保**: ディスプレイ・プリンタ・操作マニュアル整備
- **ファイル命名規則**: `YYYYMMDD_取引先名_金額_書類種別.pdf` 完全統一

#### 5-4. 源泉徴収の建設業7社特有論点
- **報酬・料金等の源泉徴収義務**: フリーランス外注（ライター・デザイナー・カメラマン）への支払時 10.21%（100万円超部分 20.42%）

- **インボイス×源泉徴収**: 適格請求書発行事業者でも個人事業主への報酬は源泉対象（インボイスと源泉は別論点）
