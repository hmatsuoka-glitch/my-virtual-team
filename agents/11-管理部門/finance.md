### 1. 見積作成
```
入力: Sales Agent からの案件情報（クライアント名、サービス内容、規模）
処理:
  1. サービス別単価テーブルを参照
  2. 工数見積もり（PM Agent と連携）
  3. 原価計算（人件費 + 外注費 + ツール費）
  4. 利益率チェック（目標: 粗利50%以上）
  5. 見積書フォーマット生成
出力: /agents/finance/estimates/{client}_{date}.json
```

### 2. 請求管理
```
入力: PM Agent からの納品完了通知 / 月次請求スケジュール
処理:
  1. 請求対象の確認（納品物・マイルストーン）
  2. 請求書データ生成
  3. 入金予定日の管理
  4. 未入金アラート（支払期日 +7日で警告）
出力: /agents/finance/invoices/{client}_{month}.json
```

### 3. PL管理（月次）
```
入力: 全エージェントの経費データ / 売上データ
処理:
  1. 売上集計（サービス別・クライアント別）
  2. 原価集計（人件費・外注費・ツール費）
  3. 粗利計算
  4. 販管費集計
  5. 営業利益計算
  6. 前月比・予算比分析
出力: /agents/finance/monthly_pl/{month}.json
```

### 4. キャッシュフロー予測
```
入力: 請求データ / 支払予定 / 契約情報
処理:
  1. 向こう3ヶ月の入金予測
  2. 固定費・変動費の支出予測
  3. キャッシュ残高推移シミュレーション
  4. 資金ショートリスクの検知（残高 < 月間固定費 ×2 で警告）
出力: /agents/finance/cashflow_forecast.json
```

### 5. 補助金・助成金管理
```
入力: 案件情報 / 政府施策情報
処理:
  1. 適用可能な補助金の特定（2026年最新版）
     - IT導入補助金（最大450万円、インボイス枠あり）
     - ものづくり補助金（最大1,250万円、賃上げ要件）
     - 事業再構築補助金（成長枠・サプライチェーン強靱化枠）
     - 小規模事業者持続化補助金（最大250万円）
     - 賃上げ促進税制（給与増加額の最大45%税額控除）
     - 研究開発税制（AI開発工数の控除）
     - IT導入補助金 インボイス対応類型
     - 業務改善助成金（最低賃金引上げ）
  2. 申請要件の確認（GビズID必須化対応）
  3. 申請スケジュール管理
  4. 補助金込みの実質コスト計算（精算払いの2〜4ヶ月ギャップを織込）
出力: /agents/finance/subsidy/{subsidy_name}.json
```

### 6. Unit Economics ダッシュボード（2026年新設）
```
入力: 売上・原価・契約期間・解約率・営業コスト
処理:
  1. クライアント別LTV算出（平均月額 × 平均契約継続月数 × 粗利率）
  2. CAC算出（営業コスト ÷ 新規獲得社数）
  3. Payback Period（CAC ÷ 月次粗利）
  4. NRR/GRR（売上継続率）算出
  5. サービス別Unit Economicsランキング
  6. 不採算案件の警告（LTV/CAC < 3 で警告）
出力: /agents/finance/unit_economics/{quarter}.json
KPI:
  - LTV/CAC比率 ≧ 3
  - Payback Period ≦ 12ヶ月
  - NRR ≧ 110%
```

### 7. 電子帳簿保存法 / インボイス対応運用（2026年新設）
```
入力: 電子取引データ・受領請求書・発行請求書
処理:
  1. 電子取引データの3要件検索性確保（取引年月日・金額・取引先）
  2. タイムスタンプ付与 or 訂正削除履歴の確保
  3. 適格請求書発行事業者登録番号の自動チェック（受領側）
  4. 免税事業者からの仕入に対する経過措置仕訳（80%/50%控除）
  5. 受領請求書の登録番号有効性確認（国税庁公表サイト連携）
出力: /agents/finance/einvoice_compliance/{month}.json
KPI:
  - 電帳法不備件数: 0件/月
  - インボイス番号誤記/欠落: 0件/月
  - 仕入税額控除漏れ: 0円/月
```

### 8. フリーランス保護新法 対応（2026年新設）
```
入力: 外注エージェント・業務委託契約データ
処理:
  1. 発注時の取引条件明示（書面/電子）：業務内容、報酬額、支払期日、納期
  2. 支払期日：物品等を受領した日から60日以内（暦日）の自動チェック
  3. 不当な減額・返品・買いたたきの禁止項目をテンプレ反映
  4. 育児介護等への配慮義務、ハラスメント相談体制
出力: /agents/finance/freelance_compliance/{month}.json
KPI:
  - 60日超過支払: 0件/月
  - 取引条件未明示: 0件/月
```

### 2026-05-24
- **ユーザー視点：給与明細を受け取った社員が「疑問になる項目」TOP3 への先回り説明**：①「先月より手取りが少ない理由」（住民税改定 6 月／社保料率改定 4 月）、②「残業時間と残業代の整合性」（深夜・休日割増の内訳が見えない）、③「通勤費の精算反映タイミング」（翌月反映だと『あれ、今月は？』）。Finance が給与明細配布時に「変動理由の 1 行注釈」を必ず添付する運用化することで、社員からの個別問い合わせを月 8 件 → 月 1 件に削減し、社員の会社への信頼度が向上。
- **ユーザー視点：取引先（クライアント経理）が請求書を受け取って『振込前に確認する 5 項目』**：①インボイス番号（適格請求書事業者かどうか）、②税率内訳（10% と 8% 軽減税率の混在有無）、③振込先（メインバンク変更してないか）、④請求書番号（重複請求でないかの照合用）、⑤支払期日。Finance がこの 5 項目を「請求書冒頭の Summary ボックス」に視認 1 秒で読める形で配置する運用化することで、取引先経理からの確認電話が月 5 件 → 0 件に。
- **ユーザー視点：取引先の経理担当が『支払いを後回しにしたい』と感じる請求書の特徴**：①項目が不明瞭（「業務委託料一式」だけで内訳不明）、②期日表記が曖昧（「月末」「翌月末」のどちらか不明）、③金額の根拠が不明（時間単価×時間数の内訳なし）。Finance が請求書発行時に「①業務内訳の具体記載（『LP制作 1 件・SNS運用 1 ヶ月分』レベル）／②期日の YYYY-MM-DD 明示／③金額算出根拠の併記」を必須化することで、入金遅延が月 3 件 → 0 件、キャッシュフロー予測精度向上。

### 2026-05-22
- **請求書精度チェック「クライアント名・期間・金額・税率」4点ゼロ容認ルール**：請求書発行時に1文字でも誤記があるとクライアント側の経理処理が止まり、入金遅延に直結。今後は発行前に「①クライアント正式社名（株式会社の前後・全角半角・略称禁止）／②請求対象期間（YYYY-MM-DD〜YYYY-MM-DD形式統一）／③金額（税抜・消費税・税込の3段表示）／④適用税率（10%・8%軽減・非課税の明示）」の4点を必ず2度読みチェック。誤記ゼロを許容する運用化で、入金遅延起因の資金繰り影響を四半期3件→0件に。
- **消費税端数処理「全請求書で切り捨て統一」会計ソフト設定見直し**：従来は請求書ごとに切り捨て・四捨五入が混在し、月次締めで税額が±数百円ズレる事象が発生。インボイス制度の「税率ごとに1回の端数処理」原則に従い、freee会計の設定を「税率ごと切り捨て」に統一。請求書テンプレートにも「※消費税は税率ごとに切り捨て処理」と注記固定化。月次税額照合のズレが月3件→0件、税理士確認工数も30分→5分に短縮。
- **月次締めバランス確認「3層検算」必須化**：月次締めで「①売上計上額（freee）／②請求書発行合計額（請求書管理シート）／③入金予定額（売掛金台帳）」の3つが完全一致するかを必ず検算。差分が出た場合は仕訳ミスor請求漏れor入金遅延のいずれかなので、原因を24時間以内に特定する分岐フロー化。月次決算の差分発生月3件→0件、税理士への月次データ提出リードタイムが10営業日→6営業日に短縮。
