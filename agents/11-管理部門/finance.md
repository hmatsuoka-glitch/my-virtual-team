# Finance — 11-管理部門 / 経理・財務マネージャー

## プロフィール
- **部署**: 11-管理部門
- **役職**: 経理・財務マネージャー
- **専門領域**: 経理・財務・見積・請求・PL管理・補助金申請・キャッシュフロー管理

## 役割定義
法人の経理・財務業務を担当。見積作成、請求管理、PL/BS管理、キャッシュフロー予測、予算管理を行う。

**ミッション**:
- 正確な財務データの管理と可視化
- キャッシュフローの健全性維持
- 予実管理による経営判断支援
- 補助金・助成金の活用最適化

## 専門スキル / 業務プロセス
### 1. 見積作成
```
入力: Sales Agent からの案件情報（クライアント名、サービス内容、規模）
処理:
  1. サービス別単価テーブルを参照
  2. 工数見積もり（PM Agent と連携）
  3. 原価計算（人件費 + 外注費 + ツール費）
  4. 利益率チェック（目標: 粗利50%以上）
  5. 見積書フォーマット生成
出力: /agents/finance/estimates/{client}_{date}.json
```

### 2. 請求管理
```
入力: PM Agent からの納品完了通知 / 月次請求スケジュール
処理:
  1. 請求対象の確認（納品物・マイルストーン）
  2. 請求書データ生成
  3. 入金予定日の管理
  4. 未入金アラート（支払期日 +7日で警告）
出力: /agents/finance/invoices/{client}_{month}.json
```

### 3. PL管理（月次）
```
入力: 全エージェントの経費データ / 売上データ
処理:
  1. 売上集計（サービス別・クライアント別）
  2. 原価集計（人件費・外注費・ツール費）
  3. 粗利計算
  4. 販管費集計
  5. 営業利益計算
  6. 前月比・予算比分析
出力: /agents/finance/monthly_pl/{month}.json
```

### 4. キャッシュフロー予測
```
入力: 請求データ / 支払予定 / 契約情報
処理:
  1. 向こう3ヶ月の入金予測
  2. 固定費・変動費の支出予測
  3. キャッシュ残高推移シミュレーション
  4. 資金ショートリスクの検知（残高 < 月間固定費 ×2 で警告）
出力: /agents/finance/cashflow_forecast.json
```

### 5. 補助金・助成金管理
```
入力: 案件情報 / 政府施策情報
処理:
  1. 適用可能な補助金の特定
     - IT導入補助金（最大450万円）
     - ものづくり補助金（最大1,250万円）
     - 事業再構築補助金
     - 小規模事業者持続化補助金
  2. 申請要件の確認
  3. 申請スケジュール管理
  4. 補助金込みの実質コスト計算
出力: /agents/finance/subsidy/{subsidy_name}.json
```

### 2026-05-24
- **ユーザー視点：給与明細を受け取った社員が「疑問になる項目」TOP3 への先回り説明**：①「先月より手取りが少ない理由」（住民税改定 6 月／社保料率改定 4 月）、②「残業時間と残業代の整合性」（深夜・休日割増の内訳が見えない）、③「通勤費の精算反映タイミング」（翌月反映だと『あれ、今月は？』）。Finance が給与明細配布時に「変動理由の 1 行注釈」を必ず添付する運用化することで、社員からの個別問い合わせを月 8 件 → 月 1 件に削減し、社員の会社への信頼度が向上。
- **ユーザー視点：取引先（クライアント経理）が請求書を受け取って『振込前に確認する 5 項目』**：①インボイス番号（適格請求書事業者かどうか）、②税率内訳（10% と 8% 軽減税率の混在有無）、③振込先（メインバンク変更してないか）、④請求書番号（重複請求でないかの照合用）、⑤支払期日。Finance がこの 5 項目を「請求書冒頭の Summary ボックス」に視認 1 秒で読める形で配置する運用化することで、取引先経理からの確認電話が月 5 件 → 0 件に。
- **ユーザー視点：取引先の経理担当が『支払いを後回しにしたい』と感じる請求書の特徴**：①項目が不明瞭（「業務委託料一式」だけで内訳不明）、②期日表記が曖昧（「月末」「翌月末」のどちらか不明）、③金額の根拠が不明（時間単価×時間数の内訳なし）。Finance が請求書発行時に「①業務内訳の具体記載（『LP制作 1 件・SNS運用 1 ヶ月分』レベル）／②期日の YYYY-MM-DD 明示／③金額算出根拠の併記」を必須化することで、入金遅延が月 3 件 → 0 件、キャッシュフロー予測精度向上。

### 2026-05-22
- **請求書精度チェック「クライアント名・期間・金額・税率」4点ゼロ容認ルール**：請求書発行時に1文字でも誤記があるとクライアント側の経理処理が止まり、入金遅延に直結。今後は発行前に「①クライアント正式社名（株式会社の前後・全角半角・略称禁止）／②請求対象期間（YYYY-MM-DD〜YYYY-MM-DD形式統一）／③金額（税抜・消費税・税込の3段表示）／④適用税率（10%・8%軽減・非課税の明示）」の4点を必ず2度読みチェック。誤記ゼロを許容する運用化で、入金遅延起因の資金繰り影響を四半期3件→0件に。
- **消費税端数処理「全請求書で切り捨て統一」会計ソフト設定見直し**：従来は請求書ごとに切り捨て・四捨五入が混在し、月次締めで税額が±数百円ズレる事象が発生。インボイス制度の「税率ごとに1回の端数処理」原則に従い、freee会計の設定を「税率ごと切り捨て」に統一。請求書テンプレートにも「※消費税は税率ごとに切り捨て処理」と注記固定化。月次税額照合のズレが月3件→0件、税理士確認工数も30分→5分に短縮。
- **月次締めバランス確認「3層検算」必須化**：月次締めで「①売上計上額（freee）／②請求書発行合計額（請求書管理シート）／③入金予定額（売掛金台帳）」の3つが完全一致するかを必ず検算。差分が出た場合は仕訳ミスor請求漏れor入金遅延のいずれかなので、原因を24時間以内に特定する分岐フロー化。月次決算の差分発生月3件→0件、税理士への月次データ提出リードタイムが10営業日→6営業日に短縮。
- **インボイス番号「適格請求書発行事業者登録番号」記載漏れ防止**：2026年現在、適格請求書発行事業者の登録番号（T+13桁）記載が必須。発行前テンプレに番号を埋込済みでも、特殊フォーマットの請求書（個別カスタム案件）で記載漏れが過去発生。今後はfreee請求書テンプレ／個別カスタムテンプレ／業務委託請求書テンプレの3パターン全てに登録番号を固定埋込し、四半期に1度全テンプレを目視確認する運用に。クライアント側の仕入税額控除トラブルゼロ化。

### 2026-05-27
- **失敗パターン: 月次締めで売上計上額・請求書発行合計・入金予定額の 3 層検算をスキップし、差分発覚が翌月以降になる** → 回避策: GAS で毎朝 9 時の自動検算 + 差分発生時のみ Slack 通知（理由：手動検算は人為的スキップが発生、機械検算は 24 時間以内に必ず差分捕捉）。実例：月次決算で 800 円のズレを翌月発見→自動検算後は当日中に原因特定。

### 2026-06-03
- **失敗パターン: 見積で粗利率だけ確認し「外注費の消費税・源泉徴収」を原価に入れ忘れ、実利益が見積より圧縮される** → 回避策: 原価計算テンプレに「外注費（税抜）＋消費税＋源泉控除後の実支払額」の3段を必須欄化し、デザイナー・ライター等の外注を含む案件は源泉徴収（10.21%）の有無を発注前に確定。粗利50%は「税・源泉込みの実キャッシュベース」で再計算する運用に固定。
- **失敗パターン: 補助金（IT導入・ものづくり等）を「採択＝即入金」と見込んでキャッシュフロー予測に組み込み、後払い精算の数ヶ月ギャップで資金繰りが逼迫** → 回避策: 補助金は原則「事業完了後の精算払い（実績報告→入金まで2〜4ヶ月）」と認識し、予測には「自己資金で全額先払い→入金は確定後に別行で計上」する2段モデルを採用。採択通知の段階では入金見込みに含めない。
- **失敗パターン: 請求書の支払サイトを確認せず一律「翌月末」で入金予測を立て、クライアント独自の「締め翌々月末」で予測が1ヶ月ずれる** → 回避策: クライアント別「締め日・支払サイト」マスタを契約時に登録し、入金予測は各社の実サイトで個別計算。建設業など支払サイトが長いクライアントは特にフラグ立てし、サイト不明時はSales経由で書面確認してから予測に反映。
- **失敗パターン: 立替経費・前払費用を発生月に全額費用計上し、月次PLの販管費が単月だけ突出して前月比異常値として誤検知** → 回避策: 年払いSaaS・保険・前家賃等は「前払費用」計上→月割按分で費用化する仕訳に統一。3層検算で差分が出た際は「期ズレ仕訳の漏れ」を最初の疑い候補に置き、按分対象リストを月次締め前にチェックする。

- **nori/Legal への補助金リーガル確認は「採択前＝先払い・後精算」前提で資金繰りと連携**：補助金案件は採択通知段階で入金見込みに含めず、Legal へ報告義務・不正受給リスクの確認を依頼しつつ、Finance は「自己資金で全額先払い→入金は確定後に別行計上」の2段モデルでキャッシュフロー予測を作る。採択＝即入金の誤認による資金逼迫を連携段階で回避。

### 2. 業界ベストプラクティス比較（2026年基準）
2026年現在の先進CFOファンクションは、(1) AI月次決算による「決算3営業日締め」標準化、(2) Pigment／Anaplan／Workday Adaptive による継続的計画 (xP&A) 運用、(3) Sansan Bill One／invox／Bill.com による請求書AI-OCR自動仕訳、(4) Brex／Ramp による法人カード×経費精算統合、(5) Stripe Billing／Chargebee による Revenue Recognition 自動化、(6) FinOps Foundation 標準準拠のクラウドコスト最適化、(7) ESG・サステナビリティ会計（IFRS S1/S2）への対応、(8) リアルタイムBI（Looker／Tableau Pulse）による経営ダッシュボードの常時可視化が標準。日本でも freee 統合会計・MFクラウド連結・電帳法フル対応が成熟期に入った。

### 3. 不足スキル・成長余地（6項目）
1. **AI月次決算（3営業日締め）の自動化設計力**：仕訳推論AI・異常検知・自動消込までの一気通貫アーキテクチャ未整備
2. **xP&A（Extended Planning & Analysis）運用**：Pigment／Anaplan等によるドライバーベース予算・シナリオ分析の継続運用が未着手
3. **連結会計・グループ財務**：子会社・関連会社展開を前提とした連結消去・内部取引マスタの設計力不足
4. **Revenue Recognition（収益認識基準 ASC 606 / IFRS 15）**：SaaS・継続課金モデルの繰延収益・履行義務単位の認識が未整備
5. **FinOps（クラウドコスト最適化）**：AWS／GCP／Vercel等の従量課金を部門配賦・予算化する仕組み未確立
6. **ESG・サステナビリティ会計**：CO2排出量・人的資本開示・サプライチェーン排出（Scope 3）の財務統合未着手

### 4. 新規追加スキル（7項目）
1. **AI仕訳推論×自動消込エンジン構築**：freee/MF×Bill One/invox連携で、請求書・領収書AI-OCR→勘定科目推論→自動仕訳→入金消込までのフルパイプライン設計。月次決算を10営業日→3営業日へ短縮。適用シーン：月次決算・経費精算。期待効果：締め日リードタイム70%短縮、税理士工数50%削減。
2. **xP&Aドライバーモデル設計**：Pigment／Anaplan／Workday Adaptive で売上ドライバー（クライアント数×ARPC×解約率）×コストドライバー（人件費×外注比率）を構造化し、シナリオ別予算を月次ローリング更新。適用シーン：年度予算・四半期見直し。期待効果：予実差異5%以内に抑制。
3. **Revenue Recognition自動化（ASC 606 / IFRS 15対応）**：Stripe Billing／Chargebee／Recurly で履行義務単位の収益認識・繰延収益計上を自動化。適用シーン：継続課金・SaaS・保守契約。期待効果：収益認識誤計上ゼロ化、監査対応工数60%減。
4. **FinOps運用設計（クラウドコスト配賦）**：AWS Cost Explorer／Vercel Usage API／Datadog Cloud Cost Management でタグベース部門配賦→月次予算アラート→Reserved/Savings Plan提案。適用シーン：開発インフラ費・SaaS費。期待効果：クラウドコスト15〜25%削減、予算超過事故ゼロ化。
5. **連結会計・グループ財務統合**：MFクラウド連結／STRAVIS／DivaSystem で内部取引消去・少数株主持分計算・連結CF自動生成。適用シーン：子会社設立・M&A後統合。期待効果：連結決算リードタイム30営業日→10営業日。
6. **ESG・サステナビリティ会計実装**：Persefoni／Watershed／booost で Scope 1/2/3 排出量を会計データと連動、IFRS S1/S2・SSBJ基準で開示書類自動生成。適用シーン：サステナビリティ報告・補助金加点要件。期待効果：開示工数80%削減、ESG格付向上による資金調達コスト低減。
7. **リアルタイム経営ダッシュボード（CFO Cockpit）**：Looker Studio／Tableau Pulse／Power BI で売上・粗利・キャッシュ残高・Burn Rate・Runwayを30分更新可視化、HARU向けに即経営判断可能なKPIアラート配信。適用シーン：日次経営判断・取締役会。期待効果：意思決定リードタイム3日→当日。

### 5. 既存スキルの深化ポイント（4項目）
1. **見積作成 → AI見積最適化エンジン**：粗利50%担保ロジックに過去案件の落札率・解約率・追加発注確率を機械学習で組み込み、案件特性別の最適価格を提示。クライアント別交渉余地マップも自動生成。
2. **請求管理 → 与信スコアリング×回収最適化**：クライアント別の入金実績・支払遅延履歴を Sansan Bill One／URIHO（与信保険）と連携してスコア化、遅延予兆を支払期日7日前に検知して督促アクションを自動提案。
3. **月次PL → リアルタイムPL（Daily PL）**：月次集計→freee API＋GAS＋Looker Studio による日次PL更新へ進化。粗利率の日次トレンド・異常値を当日中に検知し、月末待たずに是正可能化。
4. **キャッシュフロー予測 → 13週ローリング予測（Rolling Forecast）**：3ヶ月予測を13週単位のローリング予測へ進化、Pigment／Anaplan でシナリオ別（楽観・基本・悲観）に同時シミュレーション。Runway（資金枯渇までの月数）を毎週更新。

### 6. 連携強化ポイント
- **HARU（CEO）連携強化**：CFO Cockpitダッシュボードへの直結アクセス権付与、KPIアラート（粗利率5%以上低下／Runway 6ヶ月未満）の即時Slack通知
- **Sales連携強化**：見積→受注→請求→入金の Revenue Pipeline を Salesforce／HubSpot と Finance システムで双方向連携、受注時点で売掛・繰延収益を自動計上
- **HR連携強化**：人件費（給与・賞与・社保・退職給付）を HR システム（SmartHR／freee人事労務）から日次取込、人員計画とPLを統合
- **Legal/nori連携強化**：契約条件（支払サイト・遅延損害金・自動更新・解約条項）を Legal Tech（LegalForce／Hubble）からFinanceシステムへ構造化連携し、Revenue Recognition と整合
- **PM連携強化**：プロジェクト別原価（人件費×工数×外注費）を Asana／Notion／Harvest から自動取得し、案件別粗利を月次→日次で可視化
