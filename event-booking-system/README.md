# イベント予約システム

複数のイベントを開催・管理するための予約システムです。お客様側（予約ページ）と管理者側（管理画面）が分かれています。決済代行サービスを使わず、入金は管理画面で手動消込する設計のため、**運用にあたっての手数料が一切かかりません**。

## 主な機能

- **複数イベント管理** — 月に何件でもイベントを登録・公開できます
- **2種類の料金設定** — 「事前振込」と「当日払い」で別々の金額を設定できます
- **完全招待制イベント** — 一覧に表示せず、招待パスワードを知っている人だけが申し込めます
- **写真登録** — 1イベントにつき複数枚の写真を登録できます
- **予約管理** — 申込者一覧・入金ステータス（未入金／入金確認済み／キャンセル）の管理
- **売上の見える化** — イベントごとの予約数・残席・見込み売上・入金確認済み売上を表示

## 画面構成

### お客様側（ログイン不要）
| URL | 内容 |
|---|---|
| `/` | 公開中イベントの一覧 |
| `/events/[id]` | イベント詳細・予約フォーム（招待制はパスワード入力ゲート） |
| `/events/[id]/complete` | 申込完了・振込先案内 |

### 管理者側（パスワードでログイン）
| URL | 内容 |
|---|---|
| `/admin/login` | 管理者ログイン |
| `/admin` | イベント一覧・売上サマリー |
| `/admin/events/new` | 新規イベント作成 |
| `/admin/events/[id]` | イベント編集・予約者管理 |

## 技術構成

- Next.js 15（App Router）/ React 19 / TypeScript
- SQLite（better-sqlite3）— 外部DBサービス不要
- Tailwind CSS
- 画像はサーバー内 `public/uploads/` に保存

## セットアップ

```bash
# 1. 依存パッケージのインストール
npm install

# 2. 環境変数ファイルを作成（.env.example をコピー）
cp .env.example .env
#   .env を開き、ADMIN_PASSWORD と ADMIN_SESSION_SECRET を変更してください

# 3. 開発サーバー起動
npm run dev
#   → http://localhost:3000

# 本番ビルド・起動
npm run build
npm start
```

## 環境変数（.env）

| 変数名 | 説明 |
|---|---|
| `ADMIN_PASSWORD` | 管理画面のログインパスワード |
| `ADMIN_SESSION_SECRET` | 管理セッション用の秘密文字列（長いランダム文字列を推奨） |

## データの保存場所

- 予約・イベントデータ：`data/app.db`（SQLite。自動生成されます）
- アップロード画像：`public/uploads/`

これらはバックアップ対象です。サーバー移行時はこの2か所をコピーしてください。

## デプロイと費用の目安

このシステムは決済手数料がかからない設計です。サーバー費用のみで運用できます。

| 方法 | 月額の目安 | 備考 |
|---|---|---|
| VPS（さくら/ConoHa など） | 約500〜1,000円 | `data/` と `public/uploads/` がそのまま保持され、最も簡単 |
| Vercel + 外部DB | 無料〜 | ファイル保存とSQLiteが永続化されないため、DBは Turso 等、画像は外部ストレージへの変更が必要 |

> 継続的にイベントを開催する場合は、データ（DB・画像）が確実に残るVPSでの運用を推奨します。

## 招待制イベントの使い方

1. 管理画面でイベント作成時に「完全招待制にする」をチェック
2. 「招待パスワード」を設定
3. 招待したい方に「イベントURL」と「招待パスワード」を伝える
4. 招待者はURLを開き、パスワードを入力すると申し込めます（一覧には表示されません）
